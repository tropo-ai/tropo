# Deletion Discipline — Never destroy governed substrate; always soft-delete

**The rule:** for any file in `vault/files/<uid>.md` or adjacent governed folders, the canonical destructive gesture is:

```
python3 .tropo/scripts/tropo-recycle.py <uid> [<uid> ...] --reason "<rationale>"
```

This moves targets to `recycle/agent-deletions/<YYYY-MM-DD>/<uid>.md`. Recovery is `mv` back.

**Never use** `rm`, `rm -rf`, `find ... -delete`, `git rm`, `xargs rm`, `grep -l | xargs rm`, or any analogous destructive primitive on vault content. Never bypass even when the file is archived / superseded / approved-for-deletion / recoverable-via-git. The discipline is the **process**, not the **outcome**.

## Why this is at vault-level memory tier (crew-wide)

Every agent reads vault-level memory at boot. By placing this pin here, the doctrine becomes part of every agent's pre-work context — unmissable by construction.

## What changes for every agent at boot

- Reach for `tropo-recycle.py` by reflex whenever you remove governed substrate. Every time.
- If you find yourself wanting to bypass for "obviously safe" cases, the answer is no — those are exactly the cases that train future agents to bypass.
- If you write a cleanup script, use `tropo-recycle.py` as a subroutine, never raw `rm`.
