# ADR-003: Environment Configuration for Vault Path Resolution

**Date:** March 22, 2026
**Status:** Accepted
**Authors:** G7.2 (discovery), Vela V3 (formal)
**Triggered by:** G7.2 writing its research report to the wrong vault location due to an unresolvable relative path

---

## Context

Argo-OS documents (activation files, guides, crew log, ADRs) use relative paths throughout — `agents/...`, `context/...`, etc. These paths are written relative to an assumed vault root (`Personal Vault`), but that root was never formally declared anywhere.

When G7.2 activated in a new Cowork session, it had no way to determine the vault root from the activation file alone. It wrote its report to an `` folder at the Obsidian library root instead of inside `Personal Vault/Argo-OS/`. The result was a stray folder that had to be cleaned up manually.

The vault root will also change when the project migrates out of `Personal Vault` (expected). We needed a single place where that change could be made once and have it propagate everywhere.

---

## Decision

Create `config/env.md` as the canonical environment configuration file for Argo-OS.

**What it declares:**
- `vault-root` — the root folder of the vault, relative to the Obsidian library root
- `argo-os` — the Argo-OS project folder, relative to `vault-root`
- `context` — the institutional memory folder, relative to `vault-root`
- The path resolution formula: `absolute path = [mount point] / [vault-root] / [relative path]`

**Current value:** `vault-root = Personal Vault`

**All activation files for task and standing children must instruct the child to read `config/env.md` before resolving any vault paths.**

---

## Consequences

**Positive:**
- One-line vault migration. When the project moves, update `vault-root` in `env.md` — every path in the system continues to work without changes.
- Children can now orient themselves reliably in any platform context (Cowork, Claude Code, future platforms).
- Explicit contract: agents no longer guess the vault root; they read it.

**Negative / constraints:**
- Activation file authors must remember to include the `config/env.md` read instruction. If they omit it, children face the same ambiguity G7.2 encountered.
- The config is text, not executable — agents must interpret it, not run it.

---

## Implementation

- `config/env.md` — created March 22, 2026 by G7.2 + Mike
- `config/` — new folder added to the vault
- `README.md` — updated by G7.2 to include `config/` in the vault map
- `guides/how-to-spawn-child.md` — updated by Vela V3 to reference `config/env.md` as vault root source of truth (Step 3)
