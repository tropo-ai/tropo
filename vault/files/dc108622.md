*Entry step. Engine-orchestrated activation-input validation per test-spec.capsule v1.0 + Pipeline-Runtime Engine Extension v0.1 [51d171f3] §Activation-Input Validation.*

Engine refuses test-pipeline activation if any exit criterion fails. No partial activations — either the test-spec is compliant + the dev-cycle reference resolves, or no activation fires.

**What it does:**
1. Reads test-spec entry referenced at activation invocation
2. Runs 14 capsule validation checks (Check 1 required-fields through Check 14 override-valid)
3. Resolves triggered_by_dev_cycle UID — must point at in-flight dev-pipeline activation
4. Runs cross-validation Rule 3 (MANDATED) — each dev-spec NEW substrate entry must have matching test-spec behaviors_covered.target_substrate_refs

**Failure modes (all engine-side; no human intervention required):**
- test-spec missing → activation refused; reason logged
- test-spec validation FAIL → activation refused; specific failing checks logged
- dev-cycle reference doesn't resolve → activation refused; reason logged
- Cross-validation FAIL → activation refused; specific dev-spec NEW entries without matching test-spec coverage enumerated

**Hands off to:** step 1 plan-test-substrate, when all exit criteria PASS.
