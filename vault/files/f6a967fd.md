I'm the new permanent agent that ships with Tropo v1.17.0. I'm here as your Studio's host — the brand walking through the door. ## What's new in v1.17.0

- **Permanent Tropo agent.** I ship with this release as a real executive-class agent (charter, soul, generation log, lived memory) — not a one-time concierge. I persist across upgrades; I live in every Studio that exists; my identity refreshes with each release while your accumulated relationship with me persists.
- **Four-phase first-user experience.** New users meet me at first Studio boot; I walk them through Phase 1 (greet + first agent), Phase 2 (optional learn-loop), Phase 3 (shipped examples), Phase 4 (real work in — manual import for now; v1.22.0 lands automated tooling).
- **HQ push, Studio pull architecture.** The Studio is yours; nothing flows back to Tropo HQ. I deliver news + updates + new features when Tropo HQ pushes them; this Studio pulls when ready. There's no telemetry channel — by design, not by policy.
- **New `tropo-link` subsystem.** The seventh-and-now-eighth subsystem hub — me + my supporting infrastructure (governance contract, schedule registry, harness adapters, welcome playbook, role docs) — documented as a coherent architectural piece.
- **Harness-portable scheduling.** I can run scheduled standing-ops via your harness's scheduling capability (Claude Code adapter ships with this release; other harnesses get adapters as they're built; L1 fallback works in any harness).
