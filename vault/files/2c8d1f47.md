## What ships

All 17 `@typescript-eslint/no-explicit-any` errors closed. Walked the three vault-rebuild scripts in `scripts/`; replaced `any` with `unknown` (+ narrowing) per CLAUDE.md's TypeScript discipline. Mike pre-committed the `scripts/rebuild-registry.ts` portion in parallel as `34c84bd` (parallel convergent thinking — exact same change shape).

### Code changes

**`scripts/rebuild-ledger.ts`** (11 errors closed):

- `parseFrontmatter` return type: `Record<string, any>` → `Record<string, unknown>` (4 occurrences via replace-all of the type alias).
- `collectEdges` rewrite: changed `(r: any) => ...` to `(r: unknown) => ...` with proper narrowing (`typeof r === 'object' && 'rel' in r && 'to' in r`), and similar for `(t: any)` → `(t: unknown) => if (typeof t === 'string' ...)`.
- `ts` (timestamp) field in collectEdges: changed implicit-any flow `fm.modified || fm.created || created` to explicit `typeof fm.modified === 'string' ? fm.modified : ...` ladder so the resulting string type satisfies `Edge.created: string`.
- `record as any` → `record as unknown as Record<string, unknown>` (double-cast required by TS because `IndexRecord` doesn't have an index signature).
- `projectTree as any[]` → `projectTree as Array<{ parent: string | null; uid: string }>` (real shape).
- `(r as any).role` → `(r as unknown as Record<string, unknown>).role` (line 672).
- `vals.forEach((v: any) ...)` → `(v: unknown) ...` with `typeof v === 'string'` guard.

**`scripts/migrate-tasks-v2-to-v3.ts`** (1 error closed):

- `readFile` return shape: `Record<string, any>` → `Record<string, unknown>`.
