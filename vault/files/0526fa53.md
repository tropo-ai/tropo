*Planning step. Orpheus authors plan substrate naming WHAT will change WHERE per tier, before any doc files get touched.*

The plan is the bridge between doc-spec (declared intent: "what doc changes are required") and author step (executed work: "what doc files get written"). Plan substrate is the substrate Orpheus signs off on as her commitment to the cycle's doc work.

**What it does:**
1. Reads doc-spec.doc_changes_required entries (validated at Step 0)
2. Authors plan substrate at workspace path with one plan-entry per change-required entry
3. Each plan-entry resolves: target tier; target path (existing file OR new-file marker); change_summary specifics (which sections of target file; what content lands; rationale if cross-tier coherence is being broken per Rule 6)
4. Plan-entries inherit voice_review_required + cross_reference_check defaults per tier (summary + subsystem default voice_review=true; spec default voice_review=false with prose-heavy section spot-check; per-tier cross-reference rigor split)
5. Orpheus reviews assembled plan + signs off (substrate-level commitment to the work)

**Failure modes:**
- doc-spec.doc_changes_required has entry with unresolvable target path AND no new-file marker → plan refuses to land; surfaced to Orpheus for amendment
- multi-tier doc-spec missing coverage for one tier → plan flags as incomplete; surfaced to Orpheus
- per-tier doc-spec with NEW substrate from dev-spec → Check 13 WARN fires at rebuild; plan can proceed but Orpheus reviews rationale

**Hands off to:** step 2 author-doc-substrate, when plan signed off.
