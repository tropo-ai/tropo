## What the Import Primitive Is

The mechanism by which a user's existing project folders + files become governed substrate that every existing Tropo primitive operates on without special-casing. A 10+ year domain expert downloading Tropo on a Tuesday with their real folder of strategy documents now has a "try" beat that operates on their actual work — not a fresh empty vault.

**The competitive sentence:** *"Drop your real folder. Your files stay yours. Governance is layered on top. Delete Tropo tomorrow and your work is exactly as you left it."* — structurally true after v1.25.0 ships (Tier 1).

## The Two-Tier Ladder

**Tier 1 — Reference + Sidecar (v1.25.0):** Source files stay where the user put them. Every governed file gets a sidecar at `<folder>/.tropo-studio/<filename>.tropo.md`. Vault projections at `vault/files/<uid>.md` are derived from sidecars. User-visible filesystem stays clean. Low commitment; reversible.

**Tier 2 — Vault-Native (Phase 2; v1.26.0+):** User says "promote folder X to vault-native" through an agent. Source files move into `vault/files/<uid>/source.<ext>` (per-UID directory). Metadata at `vault/files/<uid>/metadata.md`. Original folder structure preserved at `01-exchange/[date]-[root-folder-imported]/` as a symlink tree. Governance is structural — the substrate prevents accidental moves outside agent processes.

## Bullet-Proof Posture

Historical sidecar-tooling failures (Lightroom catalog corruption, Apple Photos opacity, digiKam edge cases) share a shape: automated heuristic hits ambiguity, makes a choice, silently loses or corrupts state. The import primitive defends differently via three structural moves:

1. **Categorize before applying** — every delta gets confidence + category; below 0.95 confidence is deferred to executive review or user resolution
2. **Audit log of every action** — every reconciler action writes to `.tropo-studio/reconciler/journal.jsonl`; bad calls are replayable
3. **Patterns in a playbook, not in code** — intelligence lives in reconcile-imports.playbook; new failure modes extend the playbook; the system gets smarter over time

Per arch-spec [2b49ba79](2b49ba79.md) §A.9.

## How to Use It

You don't, directly. Per the conversation-as-surface principle (substrate-tier Mike-lock 2026-05-13): the user expresses intent in conversation; agents invoke tools.

Say to an agent: *"I want to import folder X into Tropo"* or *"Extract Q3 Strategy back to my desktop"* — the agent invokes import-walker.py with the right command; the substrate updates; the agent reports back.

For substrate verification (any agent or Mike directly): `python3 .tropo/scripts/tropo-validate.py` runs the four new checks alongside all existing validators.
