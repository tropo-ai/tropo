*Authoring step. Orpheus writes the doc changes per plan. Talos engineering when machine-author needed (e.g., bulk-render of subsystem-hub member tables; auto-generated catalog updates).*

This is the substantive work of the pipeline. Plan substrate became words on a page.

**What it does:**
1. Reads plan substrate from Step 1
2. For each plan-entry: opens target file (or creates new at target path); writes change per change_summary at the correct section; preserves surrounding prose
3. Runs `.tropo/scripts/generate-relations-header.py` against every touched vault entry to refresh nav-block (catches member_of/refs edge changes; renders 📍 Path / 🔗 Self / ↓ Children / ↔ Siblings / 📥 Cited by sections clean)
4. Updates frontmatter on touched files: `modified:` to current date; `modified_by:` to authoring agent slug
5. Orpheus reviews authored substrate against plan; confirms each plan-entry's intended change actually landed at intended location

**Failure modes:**
- plan-entry's target path doesn't exist + plan didn't mark as new-file → Orpheus surfaces ambiguity; plan amendment OR new-file creation per Orpheus judgment
- nav-block render fails on touched file → Talos engineering surface; Step 4 will catch + flag explicitly per Check 9 EXTENDED (c)
- frontmatter merge conflict (concurrent edits to same file from another agent) → resolve per substrate write-scope discipline; surface to Mike if ambiguous

**Hands off to:** step 3 voice-review-substrate, when all plan-entries have corresponding authored substrate + nav-blocks rendered clean.
