## Decision

CAPSULE.md — the folder-level governance file — lives at `<folder>/.tropo-studio/CAPSULE.md`, not at `<folder>/CAPSULE.md`.

AGENTS.md remains visible at the folder root and points agents to the CAPSULE.md location.

---

## Context

CAPSULE.md governs the folder it lives in: its purpose, permitted content types, write access, and operating rules. It is a machine-facing schema file that travels with the folder (portable) but does not need to be visible to the average user browsing their vault.

The question: should CAPSULE.md be at the folder root (visible) or hidden in a subfolder?

---

## The Three-Tier Storage Model

This decision completes a three-tier storage model that now scales consistently from root to folder:

```
.tropo/ ← OS kernel (root level, hidden)
.tropo-studio/ ← vault admin (root level, hidden)
<folder>/.tropo-studio/ ← folder admin (per-folder, hidden)
```

Each level has a hidden system subfolder for infrastructure that should not clutter the user's view of their content.

---

## Rationale

**Portability:** CAPSULE.md travels with the folder. Moving or zipping a folder includes its `.tropo-studio/` subfolder and its governance. This preserves the "governance travels with the files" thesis.

**Clean human experience:** Users browsing their vault folders see their work — not system files. The `.tropo-studio/` prefix signals "system infrastructure, handled for you."

**Extensibility:** `<folder>/.tropo-studio/` is now a folder-level storage primitive. Future system infrastructure that belongs at the folder level — local indexes, folder-level memories, audit logs, migration state — has a natural home. The pattern anticipates innovation we cannot yet enumerate.

**Entry point preserved:** AGENTS.md remains visible at the folder root. It is the human and agent entry point that points to CAPSULE.md. The governance chain is intact — just one step deeper.

---

## The Folder Pattern (Locked)

Every governed folder in a Tropo vault follows this pattern:

```
<folder>/
 AGENTS.md ← VISIBLE — human/agent entry point, points to CAPSULE.md
.tropo-studio/
 CAPSULE.md ← HIDDEN — folder governance schema
 [future system files]
 [content files] ← VISIBLE — what the folder is actually for
```

---

## Consequences

1. **All existing CAPSULE.md files at folder roots must be migrated** to `<folder>/.tropo-studio/CAPSULE.md`. This is a v1.1 pre-ship task.

2. **All AGENTS.md files must be updated** to reference the new CAPSULE.md location.

3. **The boot playbook and any agent that reads CAPSULE.md** must be updated to look in `.tropo-studio/CAPSULE.md` within the current folder.

4. **The vault steward** must be updated to validate CAPSULE.md at the new path.

5. **The `create-governed-folder` skill** must create `.tropo-studio/CAPSULE.md`, not `CAPSULE.md` at the root.

6. **The release build** must include `.tropo-studio/` subfolders for every governed folder.
