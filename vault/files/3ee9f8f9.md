*Anti-aspiration gate. Validates touched docs' cross-references at the rigor each tier requires. Machine-mechanical for (a)+(b); script-dispatched for (c). Orpheus substrate-verifies the audit entry.*

This is where the pipeline catches drift before close. If a touched doc has broken UID citations, or a stale member_of edge to an archived parent, or a nav-block that renders empty when it shouldn't — Step 4 fails the step and surfaces the specific defect.

**What it does:**
1. Reads touched-files list from Step 2 + voice-review-notes from Step 3
2. For each touched file at Tier 1 (summary): runs (a) body-prose UID resolution sampled (light); records sample-pass/fail
3. For each touched file at Tier 2 (subsystem): runs (a) body-prose UID resolution full; runs (b) member_of frontmatter resolution full (strict on members); runs adjacency check (composes_with sampled, medium)
4. For each touched file at Tier 3 (spec): runs (a)+(b) full strict; dispatches machine_executable_script for (c) nav-block render: invokes `.tropo/scripts/generate-relations-header.py` against the file; captures rendered output; diffs against expected (Children populated if declared; Cited-by populated if refs declared; Path resolves to parent chain)
5. Authors cross-reference-audit entry at vault/files/<uid>.md (type: note, subtype: cross-reference-audit) recording per-doc PASS/FAIL + flagged broken-edges
6. Audit entry UID collected for Step 5 acceptance_evidence

**Validator extension surface:** Check 9 (existing in tropo-validate.py, currently scoped to spec-tier) EXTENDED per Phase B walk-lock Q3 to cover (a)+(b)+(c) per tier rigor split. The validator runs at rebuild AND at this step (belt-and-suspenders); rebuild-time finding is honest historical; step-time finding gates pipeline close.

**Trust level auto-with-verification rationale:** the checks are deterministic (validator + render-diff are mechanical), but the human-readable audit-entry that records the PASS/FAIL + the specific defect path is Orpheus-authored for downstream substrate-honesty + Mike-walkability of acceptance_evidence.

**Failure modes:**
- broken UID in body prose → step FAIL; specific UID + body-prose location logged; Orpheus returns to Step 2 to amend
- broken member_of edge → step FAIL; specific edge logged; Orpheus amends frontmatter (or surfaces upstream substrate-state issue if the target was archived)
- nav-block render diff non-empty → step FAIL; specific render-mismatch logged; Talos engineering surface if generate-relations-header.py issue, Orpheus amend if substrate issue
- per-tier rigor not satisfied → step FAIL; specific tier + missing check enumerated

**Hands off to:** step 5 close-activation, when audit entry shows ALL touched docs PASS at their tier's required rigor.
