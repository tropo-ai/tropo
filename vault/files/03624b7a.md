## Purpose

The specify stage accepts the work-item, frames the release, and produces the arch-specs that the build stage will execute. It is the first stage of dev-pipeline. Specify ends when every declared sub-system has a committed arch-spec instance and the release-plan is locked.

Does NOT govern: the design work that produced the work-item (that happened in the inbox pre-pipeline), or the artifact authoring within each arch-spec (that is build stage territory).

## Flow Rules

Linear. Step-0 must complete before step-1 starts; step-1 must complete before step-2 starts. Step-2 is terminal within this stage; on its completion the stage advances to build (3bd8f5b6) via this node's `next_steps:`.

## Cold-Boot Walk-Through

A pipeline-run enters the specify stage with a design-brief at `status: accepted`.

Step-0 transitions the work-item to `active` and writes an acceptance record. Step-1 produces a release-plan with release number (user-confirmed), intent, completion outcome, and sub-system list. Step-2 spawns one arch-spec per declared sub-system. When all arch-specs are committed, specify is done and execution advances to build.
