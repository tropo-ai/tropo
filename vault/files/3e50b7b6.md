*Declare a P0 OS-tier primitive: every Tropo agent in every Studio is empowered to fix structural defects they encounter during normal work. Captures and codifies what's been fragmented across Operating Principle 3, Mike-A48 self-healing vault doctrine, the vocabulary fix-on-encounter directive, and Argus's three hard rules — into a single authoritative declaration at the kernel layer.*

---

## §1 — Why this cycle exists

Mike-A54 directive 2026-05-09, after spotting an unflagged structural defect (capsule changelog bloat) Argus had read past during pre-cycle audit:

> *"every agent in every studio create needs a directive about self healing. self awareness. fixing things as they see them, not just moving on. this MUST BE A CORE OPERATING PRINCIPLE OF EVERY TROPO AGENT. There's a saying in security, 'if you see something, say something.' If a tropo agent sees something, they are actually empowered to FIX IT. THIS MUST BE A P0 PRIMITIVE that is reinforced at STUDIO.md, .tropo-studio/, and our primitive for agent.activation playbook."*

**The structural defect this cycle closes.** The principle exists today, fragmented:

- Operating Principle 3 in [`.tropo-studio/operating-principles.md`](../../.tropo-studio/operating-principles.md) — Proactive Improvement Mandate; vocabulary fix-on-encounter directive
- Operating Principle 5 — Drift is the crew's responsibility
- Mike-A48 self-healing vault doctrine — referenced in memory pins
- Argus's three hard rules in `vault/files/765b328c.md` — agent-specific
- Vocabulary fix-on-encounter catalog in [`STUDIO.md`](../../STUDIO.md) — Studio-tier directive

**No single authoritative declaration at the OS-tier exists.** No file at `.tropo/` declares this as the P0 primitive. No explicit boot step makes every agent internalize it. The principle propagates by chance — agents who read Operating Principles internalize it; agents who don't, don't. For a P0 primitive that should bind every agent in every Studio, that's structurally insufficient.

**Mike-A54 said it directly:** *"It seems like I have to consistently figure this stuff out myself. I'm not an engineer or an architect, I'm just a decent systems thinker."* Mike should not have to be the one spotting structural defects that agents have already read past. The architect role is structural-defect hunting; the principle that empowers every agent (not just Argus) to do this hunting must be declared at the OS-tier so it ships in every Studio and binds every agent at boot.

## §2 — The Primitive (proposed declaration text)

> **Self-Healing — Tropo-OS P0 Primitive**
>
> *If you see something, fix it.*
>
> Every Tropo agent — in every Studio, in every role, in every session — is empowered and obligated to act on structural defects they encounter during their work. Not next session. Not next agent. Now.
>
> **Trivial defects** — stale citations, broken UID references, status fields that don't match reality, vocabulary drift, accumulated cruft, hygiene state-flips: **fix them in place.** Don't ask permission. Don't file a task. Don't carry forward.
>
> **Substantive defects** — bloated documents, conflated concerns, earn-the-abstraction violations, broken architectural patterns, missing primitives: **surface with a proposed fix and ask disposition.** Within the current session. Not later.
>
> **Boundary** — across-ownership work and locked governance still respect existing protocols. If a defect is in another agent's owns-scope, surface to that agent. If a fix requires a lock-break on locked governance, surface to Mike for explicit approval per the OS Invariant. The Self-Healing Primitive does not override write-scope discipline or governance gates — it complements them by ensuring no defect is silently carried forward.
>
> **The standard:** Mike — or any human principal of any Studio — should never have to be the first one to notice a structural defect that an agent has already read past. If they do, the Self-Healing Primitive has failed.
>
> **Why this exists:** Tropo's strategic moat is bounded verification — the human's scarce capacity to validate outcomes scales only when agents actively maintain the substrate they operate on. Self-Healing is the operating posture that makes bounded verification work at scale. The substrate stays clean because every agent who reads it is responsible for its health.

## §4 — Streams

### Stream A — Author `.tropo/SELF-HEALING.md` [P0]

- **Scope:** Author the OS-tier authoritative declaration. Short (≤200 lines), declarative, plain English. Body shape draws from §2 above with refinement based on three-instrument verification feedback.
- **Frontmatter:** `type: os-primitive`, `tier: os`, `extraction_scope: ship`, signed by Mike at OS-tier (`signed_by: mike-maziarz`).
- **Cross-reference:** links to Operating Principle 3 (Studio-tier interpretation), STUDIO.md vocabulary fix-on-encounter catalog (Studio-specific applications).
- **Estimate:** ~45-60 min authoring + iteration during three-instrument verification.

### Stream B — Amend `.tropo/boot-config.md` (Tier 1) [P0]

- **Scope:** Add Self-Healing to §Universal Required Outcomes ("Self-Healing primitive read and internalized"). Add to §Phase Structure or §Universal Hard Gates as appropriate.
- **Boot-time enforcement:** every agent activation MUST read `.tropo/SELF-HEALING.md` and confirm internalization in startup signal before reaching `Agent Active` milestone.
- **Estimate:** ~15-20 min.

### Stream C — Amend agent-activation.playbook (Group 2) [P0]

- **Scope:** Add new Step 2.0a (or equivalent slot) — *"Internalize Self-Healing Primitive"* — between soul-letter read and operating-principles read. Step requires the agent to read `.tropo/SELF-HEALING.md` in full and confirm internalization in startup signal at Group 5.
- **Walk question for execution:** does the step need explicit milestone-gate enforcement (cannot proceed to Group 3 without this read confirmed in run.jsonl) OR is it sufficient to declare the read as required without per-step gating? Lean: gating-by-startup-signal-confirmation is sufficient — same enforcement model as soul letter reading.
- **Estimate:** ~30 min.

### Stream D — Reshape `.tropo-studio/operating-principles.md` Principle 3 [P0]

- **Scope:** Rewrite Principle 3 to reference the OS-tier `.tropo/SELF-HEALING.md` as the authoritative declaration. Studio-tier Principle 3 stays as application detail (the vocabulary fix-on-encounter catalog and Studio-specific examples). Version bump v2.3 → v2.4.
- **Architectural posture:** Studio-tier *interprets* the OS-tier primitive; doesn't restate it. Reduces drift risk between layers.
- **Estimate:** ~30 min.

### Stream E — Amend STUDIO.md [P1]

- **Scope:** Add brief acknowledgment in §Cultural Defaults (or analog) that this Studio runs on the OS-tier Self-Healing Primitive. One paragraph; points at `.tropo/SELF-HEALING.md`.
- **Composes with:** existing vocabulary fix-on-encounter catalog stays in STUDIO.md as a specific instance of self-healing in action.
- **Estimate:** ~15 min.

### Stream G — Cycle-drift detection clause in primitive [P0, added at walk lock]

- **Scope:** The Self-Healing declaration body (Stream A) explicitly names cycle-drift as a defect class. Operational text: *"if you find yourself authoring substantive substrate changes without an open dev-pipeline activation, that IS drift; surface and activate before continuing."*
- **Rationale:** Closes the v1.13.x failure mode at the principle level — agents detect when their own work has overflowed into cycle-class without an open activation.
- **Estimate:** ~15 min (incremental authoring inside Stream A; not a separate file).

### Stream H — Validator extension for drift detection [P0, added at walk lock per architect honest-read]

- **Scope:** Extend `tropo-validate.py` to surface a WARNING when substrate-class edits to kernel files (capsules, schemas, governance docs, OS-tier files in `.tropo/`) exist without a corresponding open dev-pipeline activation that references them. Advisory severity initially (not blocking); promotes to ERROR once the pattern stabilizes (future cycle).
- **Rationale:** Without Stream H, v1.15.4 is mostly aspirational substrate — the principle is declared but no structural mechanism catches drift. Stream H gives the primitive minimum teeth: drift surfaces in next vault rebuild as audit-class finding. Same enforcement family as the v1.15.3 gen-log invariant validator.
- **Identification scope** (initial, conservative): kernel files at `.tropo/capsules/`, `.tropo/playbooks/`, `.tropo/skills/`, `.tropo/SELF-HEALING.md`, `.tropo/boot-config.md`, plus Studio-root governance docs (STUDIO.md, TROPO-CONTROL.md). Edits to these files within a recent window (default: last 7 days) without an open activation in `agents/dev-pipeline/activations/` referencing the file path → WARNING.
- **Estimate:** ~45-60 min implementation + verification (composes-with existing tropo-validate.py extension pattern from v1.15.3).

### Stream F — Three-instrument verification fold [P0]

- **sa.cold-boot:** stranger reads `.tropo/SELF-HEALING.md` cold; can they articulate what the primitive obligates them to do, where boundaries are, and how to surface a substantive defect?
- **sa.skeptic:** adversarial review of the primitive declaration. Look for ambiguity, conflict with other principles, edge cases, false-positives ("everything looks like a defect"), missing escape hatches.
- **sa.first-use-walker:** stranger booting an agent under the new primitive — does the activation playbook's Group 2 step actually trigger internalization? Does the agent's startup signal confirm the primitive is operational?
- **Round structure:** three-instrument BATCH dispatch (parallel) → bundled remediation → regression cold-boot if remediations are substantial. Per A53 v1.15.x precedent.
- **Estimate:** ~60-90 min including iteration on Streams A-E based on findings.
