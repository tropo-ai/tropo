# agents/directors/

Registered Director agents. Directors are standing agents with persistent service roles — they do not boot for a session and retire like crew agents; they activate for specific tasks and return to standby. Examples: d.pm (project management), d.ops (operations coordinator).

---

## What Belongs Here

- **Director agent folders** (`d.<role>/`) — each director's activation files, memory, and workspace
- **`00-index.md`** — registry of all registered directors
- **`AGENTS.md`** — folder governance
- **`README.md`** — human-facing description of directors and their roles

## What Does NOT Belong Here

- Crew agents (Argus, Vela, Metis, Orpheus) → `agents/<agent-name>/`
- System agents (sa.*) → `agents/sa/`
- Visitor agents → `agents/visitors/`

## Rules

1. **One subfolder per director.** Named `d.<role>/` (e.g., `d.pm/`, `d.ops/`).
2. **Director registration via `00-index.md`.** A director not listed there is not registered.
3. **Director activation files follow the agent.capsule model** — soul, status card, and optionally a workspace.
