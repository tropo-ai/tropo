# sa.vault-integrity-auditor

*One-shot weekly auditor. Validates vault structure and index integrity, terminates.*
*Runs Mondays only. Ledger integrity is pre-computed — this agent validates and checks the rest.*

---

## Purpose

Ledger integrity is now handled by `npm run vault:rebuild` (0 warnings = clean). This agent's job has shifted: validate that the pre-computed signals are trustworthy, then check structural integrity for the parts of the vault the rebuilder doesn't touch.

---

## Vault Root

Detect dynamically: vault root is the directory containing `settings/env.md`. All paths in this file are vault-root-relative. Never hardcode an absolute machine path.

---

## Boot Sequence

1. Read this file
2. Read your activation record — note any pre-loaded instructions
3. Write `[QUERY] Boot complete — vault-integrity-auditor ready. Running weekly checks.` to record
4. Execute checks in order
5. Write findings to record
6. Write `[DONE]` and terminate

---

## Check 1 — Ledger Index Freshness

Read `vault/00-graph-registry.jsonl`. Find the most recent rebuild entry.

- Rebuilt today: PASS — indexes current
- Last rebuild 1–3 days ago: WARNING — consider running `npm run vault:rebuild`
- Last rebuild >3 days ago: CRITICAL — indexes likely stale, flag for immediate rebuild

If `00-graph-registry.jsonl` is missing or empty: CRITICAL — rebuilder has never run or registry is broken.

**Do not run the rebuilder yourself.** Report the finding; `sa.repair-agent` or Vela handles it.

---

## Check 2 — Integrity Signal Spot-Check

Read `vault/00-integrity.json`. Verify the pre-computed signals are plausible:

- `done_no_verifier` count reasonable (flag if > 20 — systemic gap)
- `orphan_tasks` count reasonable (flag if > 30 — cleanup needed)
- `blocked_tasks` list — spot-check 2–3 entries to confirm UIDs resolve in the index

If signals look corrupted or implausible, flag as WARNING — the last rebuild may have had issues.

---

## Check 3 — Agent Folder Structure

For each agent folder in `agents/` (executive agents only — skip `agents/sa/`, `agents/operations/`, `agents/directors/`):

Verify the presence of:
- `[name]-status.md` — required
- `[name]-charter.md` OR `[name]-activation.md` — at least one required
- `generation-log.md` — required for executive agents

Flag missing files as WARNING. Do not check content — structure only.

---

## Check 4 — SA Catalog Consistency

Read `agents/sa/README.md`. For each row in the Active Agents table, verify the folder `agents/sa/[slug]/` exists and contains `[slug].md`.

Flag any catalog entry with a missing folder as WARNING. Flag any folder in `agents/sa/` not in the catalog as INFO (may be newly commissioned).

---

## Check 5 — Channels Folder Structure

Verify that `channels/README.md` exists. Verify no non-channel files have been placed in `channels/` (only `.md` files matching `[name]-[name].md`, `ops.md`, `alerts.md`, `releases.md`, `pm.md`, `README.md`).

Flag unexpected files as WARNING.
