## Strategic Frame

The canonical-content-doctrine you locked at A68 + the website-as-canonical reframe you locked with me earlier today land structurally here. Articles live in the vault — they always have. Until now, they've existed as drafts going through publish-pipeline activations targeting Substack as the publish surface. With this deploy, tropo-ai.com/agentic-builders/ becomes the canonical render of those articles. Substack + LinkedIn become exposure surfaces you drive manually — not subscriber funnel, not canonical mirror, not automation we build.

Three things flip with this deploy:

1. **Subscriber relationship.** Today: Substack owns the subscriber list, the email delivery, the unsubscribe flow. After this deploy: Tropo (via Resend Audience) owns the subscriber list; Resend handles delivery, bounces, compliance. Owned distribution, not platform-dependent.

2. **First impression.** Today: visitor sees a Tropo link → bounces to Substack to read → may or may not return to tropo-ai.com. After this deploy: visitor reads on tropo-ai.com → stays in the Tropo surface → encounters Studio explainers, download CTA, the rest of the hub.

3. **Publication discipline.** Today: piece state ambiguously tracked across vault drafts + publish-pipeline activations + Substack final. After this deploy: `published_at:` frontmatter field on a vault article IS the canonical publication state. One source of truth.

## Objectives

- **Ship the article rendering surface at `tropo-ai.com/agentic-builders/`** — vault articles render to web pages; canonical home for The Agentic Builders publication.
- **Ship Resend Audience + Broadcast integration** — readers subscribe on the website; new articles trigger a Broadcast send on `published_at:` flip.
- **Migrate existing pieces** (01-04 + Greg response) — add `subtype: article` to frontmatter; backfill `published_at:` for pieces that have shipped to Substack already (treat them as published canonical from this deploy ship-date forward).
- **Establish the article-layout pattern** Talos engineers — clean reading experience on web + email template that composes with marketing chrome from deploy-1. Reference designs studied; Mike eyeball gates on both surfaces.

## Scope

**In-scope:**

- Stage 0 — Article rendering route: `app/(marketing)/agentic-builders/[slug]/page.tsx` reads `vault/files/<uid>.md` for articles where `type: document` + `subtype: article` + `published_at:` populated, parses markdown body + frontmatter, renders to web page with article-layout template.
- Stage 1 — Article-layout template: Talos engineers the visual design. Reference designs studied; reading-optimal line length, hierarchical typography, image handling, code blocks (audience is technical), pull-quotes, author byline + bio, reading-time estimate, publication date visible. Mike eyeball gate.
- Stage 2 — Curriculum/index page: `app/(marketing)/agentic-builders/page.tsx` lists published articles sorted by `published_at:` DESC. Each entry: title, subtitle, published date, reading-time estimate, hero image (if specified). Subscribe surface visible.
- Stage 3 — RSS feed endpoint: `app/(marketing)/agentic-builders/rss.xml/route.ts` generates standard RSS XML from published articles. For readers who want to consume via RSS reader; also serves as the source if Mike ever wants to automate Substack syndication later.
- Stage 4 — Resend Audience integration: subscribe form on `/agentic-builders/` (and ideally inline mid-article + at footer) POSTs email to a single Resend Audience via Audiences API. Bounce handling, unsubscribe, GDPR compliance all handled by Resend.
- Stage 5 — Resend Broadcast trigger: when an article's `published_at:` flips from empty to populated, fire a Resend Broadcast to the audience with the article body rendered via React Email template (matches marketing chrome tokens). Trigger mechanism: webhook OR build-time check OR manual API call (Talos's call on the cleanest pattern given his stack).
- Stage 6 — Article frontmatter migration: scan vault for `type: document` entries that are publication-class (member_of: agentic-builders L0 or sub-streams); add `subtype: article` field. Backfill `published_at:` for pieces 01 + 04 if Mike has shipped them to Substack by deploy date (otherwise leave empty until publish-pipeline closes them).
- Stage 7 — Token + Tailwind verification (same Stage 3 gate as web-deploy-1 + deploy-2): all colors reference `--tropo-*` tokens; zero Tailwind default color utilities on the article rendering surfaces; build clean.
- Stage 8 — Mike eyeball + iteration on rendered article + email template specifically (load-bearing design moment).
- Stage 9 — Build + commit + push (Mike approval gate at step-3) + live smoke at production URLs.
- Stage 10 — Chain-progress render at ship per Metis memory pin + brief [d3f7a921](d3f7a921.md).

**Out-of-scope (deferred to subsequent web-deploys or retired):**

- **Substack syndication automation** — RETIRED at this deploy. Mike cross-posts to Substack + LinkedIn manually for exposure. The Substack RSS Pipeline brief [4e9f7a23](4e9f7a23.md) marked as deferred / re-scope candidate; revisit if manual cross-posting becomes a burden.
- Per-stream filtering (`/agentic-builders/maz/` vs `/agentic-builders/tropo/`) — flat list at v1; revisit when tutorial stream activates at v2.0.
- Per-article assets directory — for now: image URLs in markdown body reference external CDN OR are inlined; per-article `assets/` directory if needed comes later.
- Tags / categories nav — articles may carry `tags:` frontmatter; rendering doesn't filter by tag at v1.
- Comment / discussion surface — Phase 3 (Run) territory per roadmap.
- Email-archive web view (subscriber-only article archive) — Phase 2 territory.
- Author multi-profile (multiple authors with bios) — v1 single-author shape (Mike with optional crew co-credit).
- Block 6 visual map on homepage — that's web-deploy-4 (renumbered; gated on Claude Design).

## Reference Designs for Talos (article-layout + email-template inspiration)

Article-layout reference reading (study before engineering; don't pattern-copy wholesale, look at structural choices on reading experience):

- **Stratechery** (stratechery.com) — clean reading typography, generous whitespace, footnotes pattern. Tight no-distraction layout.
- **Lenny's Newsletter** (lennysnewsletter.com) — Substack-based but the article-layout discipline (hero image, clear hierarchy, inline CTAs) is studyable.
- **Lee Robinson's blog** (leerob.io) — Next.js + MDX open source; rendering pattern + typography choices.
- **Joel Spolsky's Joel on Software archive** — classic clean-reading; minimal chrome; technical-audience-shaped.

Email-template reference reading:

- **Resend's React Email components library** (react.email) — Tropo brand tokens compose with their primitives; clean rendering across major email clients.
- **Lenny's Newsletter email** — what a "publication" email feels like vs a "transactional" email; hero + body + footer + subscribe management.

Don't copy any of these wholesale. Study; pattern-copy structural choices (typography, spacing, hierarchy); produce a Tropo-voiced design that fits the marketing chrome already established. Mike eyeball gate at Deliverable #8 catches if a structural choice doesn't land.
