## Purpose

The production-build gate of build stage. Runs `npm run build` (full Next.js production build with typecheck and route generation) against the changes from step-1, then starts the production preview and confirms the changed pages render correctly. Catches a real class of bugs that dev mode hides — missing image imports, route conflicts, type errors in unused branches, environment-variable misses.

This step exists as a separate gate (not collapsed into step-1) specifically so it can't be hand-waved. If the production build fails, the deploy doesn't ship.

## Structure

Leaf step. No children. Terminal in build stage; on close, flow advances to deploy.

## Behavior

No confirmation gate. Executor runs the production build; step closes only on green build + clean preview.

**Step work:**

1. Run `npm run build` from the tropo-ai repo root.
2. **On build failure:** capture the error to `run.jsonl` (`step_paused` event with `data.reason: production-build-failure`, `data.error: <build output>`). Step pauses. Agent fixes the error in step-1 territory (the implementation files), re-runs the build. Repeat until green.
3. **On green build:** start the production preview: `npm start` (or `npm run preview` per package.json). For each page in the activation-root project's `pages_in_scope`: curl the production preview at localhost:3000/<path>, confirm HTTP 200, confirm key content strings render (one or two stable strings per page, named in advance — page title, primary heading, or known content marker).
4. **On preview failure (page 404, content missing, render error):** capture to `run.jsonl`, pause, agent fixes, re-run build.
5. **On all-green:** write `step_completed` event to `run.jsonl`. Stage advances to deploy.

**Scope:** only the pages in `pages_in_scope` are smoke-tested here. The full site doesn't get re-tested per deploy — that's deploy stage's smoke-test-live concern (against the live URL), and even there it's scoped to the changed pages.

## Inputs

- Source code from step-1.
- Activation-root project's `pages_in_scope` — determines what to verify.

## Outputs

- Confirmation that `npm run build` succeeded.
- Confirmation that the production preview renders the changed pages correctly.
- `run.jsonl` events: `step_started`, `step_paused` (on failures), `step_completed`.
