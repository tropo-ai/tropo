## Intent

A vault-ops-pipeline activation exercises vault maintenance work as a governed cycle. Two trigger modes share one definition:

- **Scheduled** - daily and weekly fleet-ops cadence. Replaces today's pattern where Vela reads fleet-ops.playbook at boot and dispatches by hand. With scheduled activation, the pipeline opens at cadence; engine runs steps; pipeline closes.
- **On-demand** - substrate-coherence cycles (ADR collision resolution, em-dash audit sweep, vault-index repair campaign), multi-channel grooming sweeps, vault-wide repair campaigns. Author a vault-ops-spec entry; activate pipeline against it.

The substantive call this pipeline embodies: **vault maintenance is governed work, not ad-hoc captain-mode improvisation.** Every closed activation has machine-checkable evidence in run.jsonl + sa.* activation-log records + post-condition substrate state. Drift in the maintenance lane stops being invisible.

Trivial Path-1 hygiene (single-file state flips, one-line typos, em-dash fix-on-encounter, single broken UID) stays inline per the [SELF-HEALING.md OS-tier P0 primitive (db0fd9b1)](../../.tropo/SELF-HEALING.md). The pipeline covers structured multi-step work with a clean "done" gate.

## Scope (what this pipeline DOES cover)

- Daily and weekly fleet-ops runs (today's [fleet-ops.playbook](../../playbooks/fleet-ops.playbook.md) becomes a pipeline activation)
- Multi-channel grooming sweeps (V53's 5277-line argus-vela sweep would be a pipeline run)
- Substrate-coherence cycles (ADR collision resolution, em-dash audit, vault-index repair)
- Vault-wide repair campaigns surfaced by sa.repair-agent or boot-drift checks

## Scope (what this pipeline does NOT cover)

- Single-file state flips, one-line typos, em-dash fix-on-encounter, single broken UID - inline per OS primitive
- Cross-lane substrate fixes belonging to other agents (route via channels, not via my pipeline)

## Activation-input contract

Per [vault-ops-spec.capsule v1.0 (8c851a7d)](../../.tropo/capsules/vault-ops-spec.capsule.md) Required Frontmatter. Engine refuses vault-ops-pipeline activation if any of:
- No vault-ops-spec entry referenced at activation time
- vault-ops-spec entry fails capsule v1.0 validation checks at ERROR ratchet level
- `run_type: scheduled` but `dispatch_list:` empty OR any entry's `currency_checked: false`
- `run_type: on-demand` but `substrate_touches:` empty OR any entry's `verification_method:` unset

## Close-time contract

Per vault-ops-spec.capsule v1.0 close invariants. Activation cannot transition status:done until:
- All dispatch_list / substrate_touches items have completion evidence in run.jsonl
- DISPATCH/COMPLETE bulletin posted to ops.md (mandatory visibility per fleet-ops.playbook Step 5 + Mike-V54 "owner doesn't get to hide" pin)
- vault-ops-spec.closed_at populated
- vault-ops-spec.acceptance_evidence populated with sa.* activation-log UIDs + run.jsonl event UIDs + post-condition substrate state UIDs

## Composition with fleet-ops.playbook (Phase 1 wrap; Phase 2 retire later)

**Phase 1 (v1.0 - this cycle):** step 3 execute-vault-ops CALLS [fleet-ops.playbook v2.2 (768c610e)](../../playbooks/fleet-ops.playbook.md) for sa.* dispatch logic. Playbook stays as dispatch primitive (knows SA spawn pattern, activation-record convention, Group A/B ordering). Pipeline adds governance shell (spec + plan + verify + close).

**Phase 2 (later cycle):** fold fleet-ops.playbook's dispatch logic directly into pipeline steps; retire the playbook. Pipeline-runtime engine can do what playbook does today; playbook only exists because there was no pipeline above it before. Deferred to keep this cycle scope bounded.

## sa.* Currency Validation (Mike-V54 directive 2026-05-27)

When sa.* ops-agents are integrated into a vault-ops-spec dispatch_list, the plan-vault-ops step validates each sa.*'s currency against current Studio reality before dispatch. Check matrix:

| Check | What |
|---|---|
| Last modified date | sa.* activation file modified within last 90 days OR explicit "stable substrate" marker |
| Capsule version alignment | sa.* declared `capsule_version:` matches current capsule version on disk |
| Spec alignment with current vault structure | output paths still valid (e.g., `shared/orientation/daily-health-report.md` still exists); registry paths resolved |
| Owner ACL current | `owner:` field matches current crew composition; `spawnable_by:` list current |
| Scope still load-bearing | sa.* domain still describes real work the current Studio needs (e.g., sa.vault-janitor scope still applies after CAPSULE v1.24 channel-discipline amendments) |

Failed currency = sa.* gets a refresh substrate-touch (separate Path 2 work-item filed) BEFORE the dispatch fires. Don't dispatch stale ops-agents.
