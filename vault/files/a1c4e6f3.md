### Six rationale-comment suppressions

Five legitimate prop-keyed-fetch sites + one SSR-safe hydration site. Each has a `eslint-disable-next-line react-hooks/set-state-in-effect` placed immediately above the offending synchronous `setState` (inside the effect body, where the rule fires — not above the `useEffect` declaration, which doesn't intercept the report):

- `app/crew/page.tsx` — load-on-mount via `refresh()` callback (the indirection foils the rule's flow analysis).
- `components/author/KBTree.tsx` — load tree when `instance` changes.
- `components/crew/BoardsView.tsx` — load board detail when `boardId` changes.
- `components/crew/ProjectsView.tsx` — load project detail when `projectId` changes.
- `components/crew/ThreadsView.tsx` — load thread detail when `threadId` changes.
- `lib/app-shell-context.tsx` — hydrate `localStorage` state on mount (browser-only API; can't run during SSR; `useState` lazy initializer would break Next.js hydration).

### Three real refactors

**`components/crew/Badges.tsx` (RelativeTime)** — was calling `Date.now()` directly in render. Replaced with `useState` + `useInterval(60_000)` so the timestamp ticks itself every minute instead of drifting silently on parent re-renders.

**`components/ChatPane.tsx` (auto-scroll)** — was using `prevMsgCount` ref + ref-write-during-render to detect message-count changes, then `requestAnimationFrame` to scroll. Replaced with `useEffect` keyed on `messages.length`, using React's own change detection. Drops the `prevMsgCount` ref entirely. The rAF stays (defers scroll until after layout so the new bubble paints first); a `cancelAnimationFrame` cleanup avoids stale scrolls on rapid streaming chunks.

**`components/author/DiffViewer.tsx` (focused-op derive)** — was using a `useEffect` to recompute `focusedId` when `reviewOneByOne` or `editSet.id` changed, with an `exhaustive-deps` disable to deliberately ignore operation-content mutations. Replaced with the canonical "Adjusting some state when a prop changes" derive-on-render pattern (React docs): track previous values via state, compare in the render body, setState in the same render. Preserves the exact triggering semantics (only re-focus on `editSet.id` swap, not on operation mutations within the same set) — sa.ui confirmed the original disable was deliberate per the long-standing comment.
