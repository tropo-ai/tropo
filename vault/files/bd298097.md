*Plan substrate authoring. For scheduled runs: derive dispatch order + re-confirm sa.* currency. For on-demand: enumerate substrate touches + verification methods + owners.*

**What it does (scheduled run_type):**
1. Reads vault-ops-spec.dispatch_list
2. Re-validates each sa.* entry's currency at plan-time per Mike-V54 directive 2026-05-27:
   - Last modified date on sa.* activation file (within 90 days OR "stable substrate" marker)
   - Capsule version alignment (`capsule_version:` matches current capsule on disk)
   - Output path validity (writes_scope paths exist; registry paths resolve)
   - Owner ACL current (`owner:` matches current crew; `spawnable_by:` list current)
   - Scope load-bearing (sa.* domain still describes real work the current Studio needs)
3. Stale sa.* gets a refresh substrate-touch filed as separate Path 2 work-item (note in 01-inbox/) BEFORE this step exits. Never dispatch stale ops-agents.
4. Resolves dispatch order per [fleet-ops.playbook v2.2 (768c610e)](../../playbooks/fleet-ops.playbook.md) Step 3: Group A parallel (daily-vault-health + vault-janitor + freshness-monitor + channel-health-monitor + vault-index-monitor + reconciler; + Monday-only weeklies); Group B after (repair-agent).
5. Writes plan to activation-root folder as `plan.md`.

**What it does (on-demand run_type):**
1. Reads vault-ops-spec.substrate_touches
2. For each touch: confirms verification_method is declared + named owner + rollback trigger if applicable
3. Resolves execution order + dependencies
4. Strips out items that should be Path 1 hygiene per Self-Healing P0 (single-flip state changes, one-line typos, em-dash audit fix-on-encounter) - pipeline doesn't carry trivial work
5. Writes plan to activation-root folder as `plan.md`

**Failure modes:**
- sa.* currency FAIL with no refresh substrate-touch filed - step refuses to exit; activation halts until refresh filed
- on-demand touch missing verification_method - same shape; halts

**Hands off to:** [step 3 execute-vault-ops (7a683e3a)](7a683e3a.md) when all exit criteria PASS.
