*Terminal step. Closes the doc-spec activation + back-feeds the dev-pipeline activation gate.*

This is where the pipeline's work becomes substrate-historical record. doc-spec entry flips to status:done with full acceptance_evidence; the cross-pipeline coupling at the dev-pipeline gate fires.

**What it does:**
1. Reads cross-reference-audit UID from Step 4 + voice-review-notes UIDs from Step 3
2. Updates the activating doc-spec entry:
   - `closed_at:` current ISO-8601 timestamp
   - `acceptance_evidence:` list populated with voice-review-notes UIDs + cross-reference-audit UID
   - `stage:` done; `state:` active
3. Runs Check 11 close-invariant from doc-spec.capsule v1.0: voice-review evidence present where required by tier; cross-reference audit present where required by tier; closed_at present
4. Updates pipeline-run.capsule instance: status:done; close timestamp recorded
5. Notifies dev-pipeline activation gate: the doc-pipeline activation UID written by engine to triggering dev-spec's `triggered_doc_activation_uids` array is now at status:done (precondition for the dev-pipeline activation close per [Pipeline-Runtime Engine Extension v0.3 (51d171f3)](51d171f3.md) §Three-Pipeline Coupling State Machine; close-check reads `_activation_uids` not `_spec_uids` — companion-pair semantics per dev-spec.capsule v1.0.1: spec UIDs identify the input contract; activation UIDs carry the runtime status:done)
6. dev-pipeline activation gate evaluates close-readiness: if ALL UIDs in `triggered_doc_activation_uids` + `triggered_test_activation_uids` are at status:done, dev-pipeline can close; else blocks until parallel test-pipeline activation closes too

**Coupling note:** doc-pipeline and test-pipeline fire in parallel (per dev-pipeline step 4.5 + step 4.6); dev-pipeline's step 6 produce-release-folder blocks until BOTH triggered activations at status:done. doc-pipeline closing alone does not unblock dev-pipeline; test-pipeline must also close.

**Failure modes:**
- Check 11 close-invariant FAIL → step does NOT close; specific missing evidence logged; Orpheus returns to Step 3 (missing voice-review) or Step 4 (missing cross-reference audit) to resolve
- pipeline-run.capsule instance unwritable → engine-side error; surface to Mike + Argus
- dev-pipeline gate notification fails → engine-side coupling break; surface to Argus per [c3dc9f00 §3] engine integration lane

**Hands off to:** terminal. doc-pipeline activation complete. Substrate-historical record sealed; dev-pipeline gate updated; downstream cross-pipeline coupling fires per engine policy.
