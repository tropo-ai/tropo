## Purpose

The push-approval gate of deploy stage. Authors a structured commit message, pauses for Mike's approval, then commits and pushes to remote. The push triggers the host's auto-deploy.

This step encodes Talos's hard rule structurally: **never push to remote without Mike's approval.** The `requires_confirmation: true` flag means the pipeline cannot accidentally push — the gate is in the substrate, not in agent memory.

## Structure

Leaf step. No children. `next_steps` points to smoke-test-live.

## Behavior

**`requires_confirmation: true`.** Executor enters the step, authors the commit message, then pauses BEFORE the git operations fire.

**Step work:**

1. **Author the commit message.** Read the activation-root project for deploy number, theme, `pages_in_scope`. Read the working-tree diff (`git diff --stat HEAD`) for files-changed. Compose a structured message:
   ```
   <theme> (web-deploy-N)

   Pages: <pages-in-scope summary>
   Files changed: <count> files, +<additions>/-<deletions>

   <optional: per-deploy notes from activation-root documents>
   ```
2. **Pause.** Write `pause_started` event to `run.jsonl` with:
   - `data.reason: "push-confirmation-required"`
   - `data.commit_message: <full message>`
   - `data.files_changed: [<list>]`
   - `data.target_branch: <branch>`
3. **Wait.** Mike reviews. He confirms with `pause_resumed` (granting push approval, optionally redirecting the commit message) or rejects with explicit `pause_resumed` carrying `data.confirmation_granted: false` — in which case the run aborts at this step and the agent fixes whatever Mike named.
4. **On approval:** executor runs `git add <files-in-scope>` (NOT `git add -A` — only the files listed in `data.files_changed`, to avoid sweeping in unrelated working-tree changes), then `git commit -m "<message>"`, then `git push origin <branch>`.
5. Write `step_completed` event to `run.jsonl`. Stage advances to smoke-test-live.

**Hooks and signing:** never skip pre-commit hooks (no `--no-verify`) or bypass signing unless Mike explicitly requests it. If a hook fails: capture to `run.jsonl`, pause, agent fixes the issue, re-runs.

## Inputs

- Activation-root project: deploy number, theme, `pages_in_scope`.
- Working-tree diff at the deploying branch.
- Mike's approval (via `pause_resumed` event).

## Outputs

- Git commit with structured message + tagged files only.
- `git push` to remote (which fires the host's auto-deploy).
- `run.jsonl` events: `step_started`, `pause_started`, `pause_resumed`, `step_completed`.
