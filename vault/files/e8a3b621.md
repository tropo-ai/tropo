## What ships

Investigated whether the 2 `react-hooks/exhaustive-deps` warnings in `ArtifactPane.tsx:155` were real bugs or false-positive class. **Verdict: not real bugs.** The original code's intent was correct; the lint plugin couldn't statically verify it. A one-variable hoist makes the intent explicit and closes both warnings with zero behavior change.

## Diagnosis

The effect at lines 144-155 (pre-deploy) used `!!streamingText` directly in the deps array:

```tsx
useEffect(() => {
  if (!loading) { setShowStream(false); return; }
  if (!streamingText) { setShowStream(false); return; }
  const timer = setTimeout(() => setShowStream(true), 3000);
  return () => clearTimeout(timer);
}, [loading, !!streamingText]);
```

Author intent — fire on the **boolean-state-transition** of `streamingText` (empty → non-empty marks "stream started"), NOT on every streaming chunk. Otherwise the 3-second timer resets on every token and the stream view never appears.

Two lint complaints:

1. **Missing dependency: `streamingText`** — the body uses `streamingText` (via `!streamingText`) but only `!!streamingText` is in deps. Lint sees `!streamingText` as a usage that's not satisfied by `!!streamingText` in deps (they're not the same identifier).

2. **Complex expression in dependency array** — `!!streamingText` is computed inline; the rule wants either a bare identifier or a hoisted variable so static analysis can verify the identity.

## Fix

Hoist `hasStream = !!streamingText` to a local variable before the effect; use it both in deps and in the body:

```tsx
const hasStream = !!streamingText;
useEffect(() => {
  if (!loading) { setShowStream(false); return; }
  if (!hasStream) { setShowStream(false); return; }
  const timer = setTimeout(() => setShowStream(true), 3000);
  return () => clearTimeout(timer);
}, [loading, hasStream]);
```

Added a comment explaining why the boolean-state semantics matter (matches the original intent).

## Why this is not a behavior change

- `hasStream` recomputes from `streamingText` on every render. The boolean value is identical.
- Effect dep-array comparison: `hasStream` toggles on the same transitions `!!streamingText` would. Effect fires on the same triggers.
- Effect body now uses `hasStream` (the same boolean truthiness check), preserving the existing early-return logic.
