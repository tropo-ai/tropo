## What ships

**New components/lib:**
- `lib/app-shell-context.tsx` — AppShellProvider + useAppShell() hook + useHeaderSlots() hook. Owns app-wide state (theme/mode/power/KB/debug) with localStorage persistence. Pages register per-page chrome (breadcrumb, primary action) via useHeaderSlots.
- `components/AppShell.tsx` — outer layout primitive; renders <Header /> + provides AppShellProvider context. Used in app/layout.tsx so every route renders inside it.

**Modified:**
- `components/Header.tsx` — rewritten. No props. Reads state from useAppShell(). Uses usePathname() to vary chrome per route (chat-bearing routes show KB/mode/power; browsing routes show just nav). Adds Crew to the nav (consistency).
- `app/layout.tsx` — wraps {children} in <AppShell> so the Header renders at the top of every route.
- `app/page.tsx` — drops local state for isDark/debugOn/mode/powerMode/kb (now in context). Drops its own <Header /> JSX. Registers "New Chat" primary action via useHeaderSlots.
- `app/projects/page.tsx` — inline header markup removed.
- `app/projects/[id]/page.tsx` — drops local state, drops <ProjectHeader /> JSX, registers breadcrumb (Projects > project name) + primary actions (Settings + New Chat) via useHeaderSlots.
- `app/author/page.tsx` — inline `.ma-header` markup removed (including the secondary "V2.0 Knowledge. Managed." tagline).

**Deleted:**
- `components/ProjectHeader.tsx` (244 lines) — functionality absorbed into the unified Header.
