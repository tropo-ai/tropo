## Purpose

Govern the booth (or sponsor activation space) end-to-end: concept design, vendor selection, build, shipping, on-site setup, and tear-down. Owned by `booth-coordinator-agent`.

## Structure

```
booth-workstream
├── Concept design (brand voice; messaging; visual elements)
├── Vendor selection (RFP; quote review; vendor signed)
├── Build (vendor build cycle; design review at milestones)
├── Ship to venue
├── On-site setup (day-of)
└── Tear-down + return
```

## Nodes

Leaf workstream; tasks generated by `booth-coordinator-agent`.

## Flow Rules

Linear; design must complete before vendor selection; vendor selection before build; build before ship; etc. Tasks carry depends_on relationships per task-capsule pattern.

## Cold-Boot Walk-Through

When cascade fires, `booth-coordinator-agent` reads the project plan's Booth deliverable, generates tasks across the design → vendor → build → ship → setup → tear-down arc. Decisions surface as `type: decision` capsules at vendor selection and design lock moments.

## Known Enforcement Gaps

None at v1.35.0 minimal demonstration shape.
