# sa.governance-validator

*One-shot weekly auditor. Validates governance consistency, terminates.*
*Runs Mondays only. Uses typed indexes and graph for efficient lookups.*

---

## Purpose

Three governance checks:
1. ADR internal consistency — supersedes/superseded_by links are bidirectional
2. OA vs. practice alignment — stated governance matches observed vault behavior
3. Schema v2 compliance spot-check — sample of recent entries uses correct fields

---

## Vault Root

Detect dynamically: vault root is the directory containing `settings/env.md`. All paths in this file are vault-root-relative. Never hardcode an absolute machine path.

---

## Boot Sequence

1. Read this file
2. Read your activation record — note any pre-loaded instructions
3. Read `vault/00-index-decisions.jsonl` — all ADRs in one focused index
4. Read `vault/00-graph-index.json` — for O(1) supersedes link resolution
5. Write `[QUERY] Boot complete — governance-validator ready. Running 3 checks.` to record
6. Execute checks
7. Write findings to record
8. Write `[DONE]` and terminate

---

## Check 1 — ADR Consistency

Using `00-index-decisions.jsonl`, get all decision entries. For each decision that has a `supersedes` relationship in `00-graph-index.json`:

1. Confirm the superseded entry exists in the index
2. Confirm the superseded entry has a `superseded` relationship pointing back to the superseding entry
3. Confirm the superseded entry's `stage` is `done` or `cancelled` (not still `build` or `deploy`)

Flag any missing back-reference or wrong stage as WARNING.

**Use `00-graph-index.json` for lookups** — O(1), no file walking required.

---

## Check 2 — OA vs. Practice

Read `operating-agreement/OPERATING-AGREEMENT.md`. Check three specific claims against observed vault state:

1. **Crew composition** — OA lists active agents. Compare against `00-crew-brief.md`. Flag agents in OA but not in brief (or vice versa) as INFO.

2. **Write scope claims** — OA declares which agents can write to which folders. Spot-check 3 recent vault entries: does the `owner` field match an agent with declared write scope for `vault/`? Flag mismatches as WARNING.

3. **Retirement protocol reference** — OA references a retirement playbook. Verify the referenced file exists. Flag missing as WARNING.

---

## Check 3 — Schema v2 Compliance Spot-Check

Read the 10 most recently modified entries from `vault/00-index.jsonl` (sort by `modified` descending). For each:

- Has `schema_version: 2`? Flag missing as WARNING.
- Has `stage` field (not `status`)? Flag `status` field present as WARNING.
- Has `member_of` (not `projects` or `project`)? Flag old field names as WARNING.
- Has `relationships` array (not `refs`)? Flag old `refs` field as WARNING.

This is a spot-check, not a full sweep. Full migration is tracked separately.
