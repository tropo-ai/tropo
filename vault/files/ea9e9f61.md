## Purpose

The runtime-first-step of app-pipeline. Accepts the human-issued deploy trigger ("run app-pipeline as app-deploy-N, theme: '<one-line>', scope: <files-or-features>"). Authors the activation-root project per pipeline.capsule Rule 10. Pauses for Mike to confirm the deploy number + theme + scope before flow advances to build.

This is the scope-confirmation pause point of the pipeline. Build and verify-build run through without pauses once specify clears; deploy has its own push-confirmation gate at step-3.

## Structure

Leaf step. No children.

## Behavior

**`requires_confirmation: true`.** When pipeline-run executor enters this step, it transitions to `status: paused` BEFORE step work begins, writes a `pause_started` event to `run.jsonl` with:

- `data.reason: "human-confirmation-required"`
- `data.proposed_deploy: <app-deploy-N>`
- `data.proposed_theme: <one-line-theme>`
- `data.changes_in_scope: [<list of file paths or feature names>]`
- `data.expected_visual_diff: <"zero" | "described" | "n/a">` — when refactoring, declaring "zero" up front is the verification commitment for verify-build + smoke-test

The run waits. Mike (the authorized confirmer per pipeline-run.capsule v1.1+) appends a `pause_resumed` event with `data.confirmation_granted_by: mike` and any redirects on number, theme, scope, or expected-visual-diff. Run resumes; step work executes.

**Step work (once confirmed):**

1. Author the activation-root project for the deploy at `vault/files/<deploy-project-uid>.md`:
   ```yaml
   type: project
   name: "app-deploy-N"   # the Mike-confirmed number
   title: "app-deploy-N — <theme>"
   status: active
   state: standing   # transitions to done at end of pipeline-run
   owner: talos
   member_of: ["2918e3b4"]   # app-pipeline root
   changes_in_scope: [...]
   theme: "<one-line>"
   expected_visual_diff: "<zero | described | n/a>"
   live_url: "<optional — when platform gains a public deploy target>"
   ```
2. Write `step_completed` event to `run.jsonl`. Stage advances to build.

## Inputs

- `deploy_version` (string, required) — e.g., `app-deploy-1`. Mike-confirmed.
- `deploy_theme` (string, recommended) — one-line description of what this deploy is about. Mike-confirmed.
- `changes_in_scope` (array of strings, required) — file paths or feature names included in this deploy. Mike-confirmed.
- `expected_visual_diff` (string, recommended) — `zero` for refactor-class, `described` (or prose) for feature-class, `n/a` for headless changes (e.g., API/lib changes with no UI surface).

## Outputs

- Activation-root project at `vault/files/<deploy-project-uid>.md` — becomes the graph parent for all downstream artifacts the run produces.
- `run.jsonl` events: `pause_started`, `pause_resumed`, `step_started`, `step_completed`.
