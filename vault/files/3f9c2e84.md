## What ships

Designed by [sa.ui record 003](../../agents/sa/sa.ui/activation-log/003-talos-t6-record.md) (third sa.ui dispatch; verdict `pass` with high-confidence mappings for 8 of 9 tokens, medium-confidence on `--arctic`). Closes T5-inherited inbox finding [53a7a624](53a7a624.md) (9 undefined CSS tokens) — the last open Path 2 finding from the UI Foundation Refactor cycle.

### Bonus discovery — two pre-existing light-theme bugs FIXED as a side effect

While auditing token usage, sa.ui surfaced two latent bugs not in the original finding:

- **`.tropo-author-treeFileMenu`** (Author file-tree dropdown) was rendering `--tropo-graphite-soft` (dark) on light theme because its fallback wasn't theme-aware.
- **`.tropo-author-confirmBox`** (Author apply-confirmation modal) was rendering `#1a1f2e` (dark-navy) on light theme for the same reason.

Both now resolve to `--tropo-surface-bg` which is theme-aware. The light-theme parity gap closes silently.

### Token mappings (eight aliases + one in-place rename)

**`app/styles/tropo-tokens.css`** — appended a `:root` alias block under a clearly-commented "Legacy-token retirement aliases" section. Theme awareness inherited from the canonical `--tropo-*` targets:

```css
:root {
  --fg:        var(--tropo-fg);
  --fg-muted:  var(--tropo-fg-faint-translucent);
  --glass:     var(--tropo-surface-glass-soft);
  --radius:    var(--tropo-radius-lg);
  --arctic:    var(--tropo-accent-soft);
  --font-mono: var(--tropo-mono);
  --surface:   var(--tropo-surface-bg);
  --glass-bg:  var(--tropo-surface-bg);
}
```

**`app/globals.css:4255`** — `var(--bg)` (semantically mis-named at the use site; meant "ink on accent," not page bg) renamed in-place to `var(--tropo-accent-ink)`. NO alias for `--bg` — the name was misleading and aliasing it would propagate the confusion.

**`app/globals.css:5073-5080`** — `.tropo-crew-tag` rule deleted. Zero JSX callers (sa.ui verified). Dead CSS gone.

### The `--arctic` resolution

Mike-resolved (via Talos T6) on sa.ui's Open Question #2: resolve to `--tropo-accent-soft` (warm palette coherence) rather than introducing a new `--tropo-link: #81B2F4` cool-blue token. Rationale: the Crew surface uses no blue elsewhere; introducing a one-off cool token is a palette-design call that warrants a deliberate cycle, not a cleanup pass. Sa.ui flagged the trade-off explicitly via git archaeology (original MindBridge intent was `#81B2F4`). Future palette expansion can add `--tropo-link` if cool-accent text becomes design-needed.
