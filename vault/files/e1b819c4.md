## Purpose

Reads the committed release-plan and authors one `arch-spec.capsule` instance per declared sub-system. Strict 1:1: one sub-system maps to exactly one arch-spec. Each arch-spec is numbered, registered, and linked back to the release-plan before this step is done.

This step is the terminal step of the specify stage. On its completion, the specify stage closes and build begins.

## Structure

## Nodes

## Flow Rules

Completes when: for every sub-system in the release-plan's sub-systems list, one arch-spec.capsule instance exists in the Vault, is registered, and is referenced from the release-plan. No sub-system is without an arch-spec; no arch-spec is without a sub-system (1:1 enforced).

Terminal within specify stage. On completion, specify stage advances to build (3bd8f5b6).

## Cold-Boot Walk-Through

The executor reads the release-plan's sub-systems list. For each sub-system, it authors one arch-spec instance: title = `<sub-system-name> — arch-spec`, member_of = this release-plan, status = draft. It registers each arch-spec in registry.yaml. It updates the release-plan with `composes_into:` references to each arch-spec. All arch-specs committed → step done → specify stage done → build begins.
