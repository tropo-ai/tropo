*Entry step. Engine-orchestrated activation-input validation per doc-spec.capsule v1.0 + Pipeline-Runtime Engine Extension v0.1 [51d171f3] §Activation-Input Validation.*

Engine refuses doc-pipeline activation if any exit criterion fails. No partial activations — either the doc-spec is compliant + the dev-cycle reference resolves + cross-validation passes, or no activation fires.

**What it does:**
1. Reads doc-spec entry referenced at activation invocation
2. Runs 12 capsule validation checks (Checks 1-10 + 12-13; Check 11 close-invariant fires at Step 5)
3. Resolves triggered_by_dev_cycle UID — must point at in-flight dev-pipeline activation
4. Resolves target_subsystem UID (or accepts explicit null with cross-subsystem rationale)
5. Runs cross-validation Check 10 (MANDATED) — each dev-spec NEW substrate entry must have matching doc-spec doc_changes_required entry OR explicit "no doc impact" rationale

**Failure modes (all engine-side; no human intervention required):**
- doc-spec missing → activation refused; reason logged
- doc-spec validation FAIL → activation refused; specific failing checks logged
- dev-cycle reference doesn't resolve → activation refused; reason logged
- target_subsystem doesn't resolve + no cross-subsystem rationale → activation refused
- Cross-validation FAIL → activation refused; specific dev-spec NEW entries without matching doc-spec coverage enumerated

**Hands off to:** step 1 plan-doc-substrate, when all exit criteria PASS.
