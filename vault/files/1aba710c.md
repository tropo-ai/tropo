## 1. What This Subsystem Covers

*Structured per subsystem-hub.capsule v1.6 §1 (In scope + Not in scope + Edge cases). Originally reframed v1.7 Stream D1 (content authority, not storage substrate); v1.8 Mike-A46 mid-cycle further narrowed scope (canonical OS documentation re-homed to NEW [tropo-documentation hub (f87e33f0)](f87e33f0.md)). v1.6 amendment converts the existing scope statement to structured enumeration.*

### In scope

- **`library/` folder content storage.** Narrative content shipped with every Tropo-OS vault. Active entries: [Studio Manifesto (efeb1ea5)](efeb1ea5.md) v1.0.3 (v1.46 authored; supersedes legacy Workshop Manifesto fbb13cca preserved at `library/the-workshop.manifesto.md`); [The Tropo Handbook (c92ae197)](c92ae197.md) (~5500 words; 11 sections; v1.4 rev 3; status:published v1.7); `library/docs/` long-form how-tos.
- **Packs (`library/packs/*`).** Extraction packs that ship with the OS: Agent Identity (Canon), Verification (Helm), Governed Work (Lantern). Plus the Green City multi-agent pilot. Container at [Packs (d9b8a9bb)](d9b8a9bb.md) (renamed from "Starter Packs" at A36/V35 deprecation 2026-04-28).
- **Crew-narrative content storage.** Captain's Log at `library/captains-log.md` (registered as governed document at v1.X by O10 task 42aca2a4); crew-status-reports; other crew-narrative substrate.
- **KB articles (typed `kb-article` at `vault/files/`).** 22+ canonical knowledge entries spanning foundational concepts (how-the-tropo-vault-works, how-projects-replace-folders, how-pipelines-work, how-playbooks-work), operational toolchain, capsule-vs-instance-naming kernel rule. *Migrated from `.tropo/kb/` at v1.19.0 per Lock A Universal Storage Convergence; navigable via subsystem hub member lists.*
- **Reading Notes substrate.** Mike's reading inputs + crew-curated reading notes (loose definition; the `library/reading-notes/` substrate referenced in subsystem-hub.capsule v1.6 drift example; v1.10 Substack Launch Style + related touchstone substrate).
- **Studio-tier content authority (NOT canonical OS docs).** v1.8 split: this hub owns *content storage substrate* (the medium); Tropo Documentation owns *canonical OS documentation* (the message). KB articles live here as substrate; the canonical-documentation thesis lives at Documentation.

### Not in scope

- **Canonical OS documentation prose.** KGAE article, primary canonical docs, README/Getting Started narrative, website surface copy, the documentation-as-subsystem thesis — all owned by [Tropo Documentation (f87e33f0)](f87e33f0.md) per v1.8 Mike-A46 split. Library stores; Documentation defines what's canonical.
- **Storage substrate machinery.** Vault files store, UID-addressing, schemaless document pattern, registry semantics, cross-vault federation primitives — [Tropo Governance (8dd772a0)](8dd772a0.md).
- **Discovery + rendering infrastructure.** Rebuilder scripts, cascade indexes, graph index, registry.jsonl, project-tree, board-render-engine, KB-render routes at `/kb/<uid>`, the website surface — [Tropo Rendering (dbc1cbbf)](dbc1cbbf.md).
- **Capsule definitions.** `.tropo/capsules/` typed primitives — governed by the meta-capsule (222873b9); operationally owned by Tropo Governance (schemas) + Tropo Playbooks (playbook-class capsules).
- **Work primitive semantics.** Owner/verifier roles, task lifecycle, project composition — [Tropo Work (2d083137)](2d083137.md).
- **Agent lifecycle + boot + memory.** Charters, soul letters, transfers, status cards, activation files, memory substrate — [Tropo Agents (99ed55fd)](99ed55fd.md).
- **Playbook orchestration + skills.** Playbook substrate, dispatch playbooks, skills layer — [Tropo Playbooks (76bab75f)](76bab75f.md).
- **ADRs + verification instruments + operating principles.** All governance-class substrate — [Tropo Governance (8dd772a0)](8dd772a0.md).

### Edge cases (multi-parent / borderline)

- **Studio Manifesto (efeb1ea5) + Tropo Handbook (c92ae197)** — long-form canonical narrative *content*, stored under `library/`. Lives in Library as content storage substrate; the canonical-content thesis (what's canonical, what supersedes what) is Tropo Documentation. Cross-subsystem composition: Library owns the medium; Documentation owns the message.
- **KB articles (`kb-article` at vault/files/)** — substrate-class entries (live here as content storage) but their primary canonical hub is Tropo Documentation per v1.8 split. Member_of edges often point to documentation hubs; storage-of-record is Library.
- **library/reading-notes/ + Substack Launch Style touchstone substrate** — content storage in Library; Mike-G56 directive locks Substack-related work as off-limits without explicit signal (substrate is preserved, not actively elaborated).
- **Captain's Log (library/captains-log.md)** — crew-narrative content storage (Library) authored across many cycles by many agents (cross-agent composition). Format + governance shape is Tropo Documentation; content storage is here.

## 5. Conscious Trade-offs

- **Schemaless document store + typed capsules.** Ledger is schemaless at the filesystem level; types are enforced by capsule definitions at check-in. Flexibility + governance without a database. Trade-off: capsule drift is possible without a mechanical validator (v1.4 target); honor-system discipline in v1.3.
- **UID addressing over path addressing.** UIDs survive renames and moves; paths don't. Trade-off: UIDs are less human-readable than paths; mitigated by the index + human-readable titles.
- **Flat `vault/files/` layout.** No subfolders; filenames are UIDs. Trade-off: filesystem isn't navigable by humans; navigation happens through collections + cascade indexes. Humans use the index + cascade; agents use the registry.

## 7. Verification Contribution (per matrix v1.0 footer)
