## Objectives

- **Ship six Studio explainer pages live at tropo-ai.com** — Vault / Agents / Capsules / Playbooks / Channels / Work — each authored as the LOCKED v1.2 Phase 1 brief Section 9 specifies.
- **Validate the "How to Build with Tropo X" register cohesion** across all six pages as a curriculum surface, not isolated explainers.
- **Honor the canonical-content-doctrine structurally** — each page routes to (a) canonical KB article for primitive depth, (b) Hello Tropo tutorial for build-through when one ships. No parallel canonical-claim prose authored on the website.
- **Establish the explainer-page template pattern** Talos can reuse for future curriculum pages (Hello Tropo series additions, deeper architectural surfaces in Phase 2).

## Scope

**In-scope:**

- Stage 0 — Page template: one Next.js component at `app/(marketing)/build-with-tropo/[slug]/page.tsx` (or equivalent) that takes per-page content props + renders the canonical layout (title + orientation copy + canonical-link block + visual slot + cross-curriculum next/prev).
- Stage 1 — Per-page content modules (6×): each page's orientation copy + canonical-link targets + visual prop + cross-curriculum links sourced from `copy.ts` extension (single source of truth for marketing strings per the pattern Talos established in web-deploy-1).
- Stage 2 — Curriculum index page at `app/(marketing)/build-with-tropo/page.tsx`: 6-page table-of-contents + brief curriculum framing + visual-map preview.
- Stage 3 — Visuals (6×): small per-page web visuals — file-tree (Vault), agent-lifecycle (Agents), capsule-instance pattern (Capsules), playbook-excerpt (Playbooks), channel-flow (Channels), pipeline-view (Work). Talos engineering proxies acceptable for v1; Mike's Claude Design refinements land in a follow-on deploy if needed.
- Stage 4 — Cross-curriculum navigation: inline next-prev links between pages + a small "Build with Tropo" sidebar curriculum index for jump-around access (collapsed by default on mobile).
- Stage 5 — Capsule page substrate-gap honesty: ship the Capsules page (Page 3) with the substrate-gap flag visible to readers — a small "Full canonical queued" note linking to the inbox-filed [c8a91f4e](c8a91f4e.md). Honors canonical-content-doctrine by not authoring parallel canonical to fill the gap.
- Stage 6 — Token + Tailwind-defaults verification (same Stage 3 gate as web-deploy-1): all colors reference `--tropo-*` tokens; zero Tailwind defaults on marketing surfaces; build clean.
- Stage 7 — Build + commit + push (Mike approval gate at step-3) + live smoke at production URLs.
- Stage 8 — Chain-progress render at ship (interim binding per Metis memory pin pending pipeline.capsule v2.7 amendment per brief [d3f7a921](d3f7a921.md)).

**Out-of-scope (deferred to subsequent web-deploys per roadmap e6f87d14):**

- Block 6 visual map ("What's in a Tropo Studio?") on the homepage (web-deploy-3 — gated on Mike's Claude Design visuals)
- Hosted-blog substrate (web-deploy-? — slot TBD; lean: insert before web-deploy-3 per Metis recommendation in board snapshot b8c4d3e2)
- Substack RSS pipeline reshape (web-deploy-4 — direction-flip amendment to brief 4e9f7a23 needed first)
- Block 5 wizard (web-deploy-5)
- GitHub Releases dynamic integration (web-deploy-6)
- Phase 2/3 surfaces (web-deploys 7+)
- Mike's full Claude Design visuals — Talos proxies ship in this deploy; Claude Design refinements are a separate follow-on candidate
