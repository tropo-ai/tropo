## Objectives

- Consolidate the platform's UI to one design-token system (`--tropo-*`), one primitives layer, and one shared shell.
- Make the next 10 platform features cheaper to build than the last one. Each shipped deploy reduces friction for what follows.
- Retire the half-finished Mo → Tropo brand migration so the codebase matches the brand.
- Keep every deploy zero-visual-diff or carefully-bounded-visual-diff: this is a refactor, not a redesign.
- Prove the three-pipeline routing (web / app / dev) by walking a real multi-deploy initiative end-to-end through `app-pipeline`.

## Scope

### In-scope

- `app/styles/tropo-tokens.css` — design tokens (extension, no removal)
- `app/globals.css` — legacy alias block, class-family selector blocks (`.mb-*`, `.ma-*`, `.crew-*`)
- `components/` — pane components migrate to consume primitives
- `components/ui/` — new primitives layer (`Panel`, `Pane`, `Chip`, `Button`, `IconButton`, `Select`, `Divider`, `Wordmark`, etc.)
- `app/layout.tsx` — shared shell placement
- All five route files (`/`, `/author`, `/projects`, `/projects/[id]`, `/crew`)
- User-visible brand strings: `"Mo"` → `"Tropo"`, storage keys, sender labels

### Out-of-scope

- Visual redesign — every change preserves current visual output unless explicitly scoped otherwise per deploy
- Backend / API changes (except where API string surfaces carry brand vocabulary — Step 4)
- Query pipeline, KB resolver, Anthropic client, intent extraction internals
- Studio governance (argo-os/) substrate (lives in dev-pipeline)
- New features or screens

## Deliverables

Eight planned deploys. Two complete; six remaining. Each deploy is one `app-deploy-N` pipeline-run. Effort estimates are rough sizing, not commitments.

### D1 — `app-deploy-1` · Token consolidation (Step 1a + 1b) ✅ DONE

- **What:** Extend `tropo-tokens.css` to coverage parity; replace legacy `:root` + `html.dark` blocks in `globals.css` with alias definitions.
- **Acceptance:** typecheck clean · build clean · 4/4 routes 200 · zero visual diff.
- **Effort:** ~1.5h.
- **Shipped:** commit `369716a` (2026-05-16). Activation root [52dadffd](52dadffd.md).
- **Self-healing finding filed:** [53a7a624](53a7a624.md) — six pre-existing undefined token refs (`--fg`, `--fg-muted`, `--bg`, `--glass`, `--radius`, `--arctic`).

### D2 — `app-deploy-2` · Sweep crew page + `.crew-*` selectors (Step 1c-crew) ✅ DONE

- **What:** Sweep 97 legacy `var(--…)` refs in `app/crew/page.tsx` and the 18 `.crew-*` selectors in `globals.css` to `--tropo-*` directly.
- **Acceptance:** typecheck clean · build clean · 4/4 routes 200 · zero visual diff · 119/119 line diff (pure 1:1 swap).
- **Effort:** ~1h.
- **Shipped:** commit `54bd51b` (2026-05-16). Activation root [f33dfb17](f33dfb17.md).

### D3 — `app-deploy-3` · Bundled Step 1c-rest + 1d (`ma` + `mb` + retire aliases)

- **What:** Sweep legacy refs in `app/author/page.tsx` + `components/author/*` + `.ma-*` selectors in `globals.css`; sweep `app/page.tsx` + `app/projects/*` + `components/{Header,ProjectHeader,ChatPane,ArtifactPane,ConfigPane,ChatHistory,ProjectSettings}.tsx` + `.mb-*` selectors in `globals.css`; then **delete** the legacy alias block in `globals.css` (`--accent`/`--bg0`/`--panel`/etc.). After this deploy, no `var(--legacy)` reference exists anywhere in the codebase.
- **Acceptance:** typecheck clean · build clean · 4/4 routes 200 · zero visual diff · `grep -rE 'var\(--(accent|bg0|bg1|panel|panel2|text|muted|border|border-glow|shadow|panel-shadow|ring|r-xl|r-lg|r-md|gap|circuit|amber|pitfall)\b' app/ components/ lib/` returns 0.
- **Effort:** ~3h (largest sweep — ~600 selectors + hundreds of var() refs).
- **Bundling rationale:** mechanical sweep, provable zero-diff, same shape as D1/D2 proved twice. Smaller per-value cost in one larger deploy than three sequential.

### D4 — `app-deploy-4` · UI primitives — Foundation (Panel, Pane, Chip, Button)

- **What:** Author `components/ui/{Panel,Pane,Chip,Button}.tsx` consuming `--tropo-*` tokens. Wire 1-2 callsites per primitive to validate the API before broad adoption.
- **Acceptance:** typecheck clean · build clean · primitive callsites visually match pre-change baseline · primitives have prop-shape stable enough to absorb existing variants (`mb-chip` chip-types, `crew-badge` badge-colors, etc.).
- **Effort:** ~3h.
- **Bundling rationale:** four primitives with related shapes — bundling keeps the API consistent across them; reviewing one without the others misses cross-cutting decisions.

### D5 — `app-deploy-5` · UI primitives — Forms + Layout (Select, IconButton, Divider, Wordmark)

- **What:** Author remaining primitives. Wire 1-2 callsites per. Migrate header-equivalent surfaces (KB select, mode select) to consume `<Select>`.
- **Acceptance:** same shape as D4. After D5, the eight planned primitives all exist + are used somewhere.
- **Effort:** ~2.5h.

### D6 — `app-deploy-6` · Shared `<AppShell>` + collapse Header / ProjectHeader

- **What:** Author `<AppShell>` (placement: `app/layout.tsx` — Next.js idiom). Holds header, nav, theme toggle, settings dropdown, user greeting. Each screen renders content inside it. Collapse `Header.tsx` (244 lines) + `ProjectHeader.tsx` (244 lines) into a single context-aware header inside `<AppShell>`. Migrate all five routes.
- **Acceptance:** typecheck clean · build clean · all 5 routes 200 · all 5 routes show consistent chrome (header, nav, theme toggle, settings) · `ProjectHeader.tsx` deleted · no header-state duplication across routes.
- **Effort:** ~4h.
- **Risk:** medium. This is the largest structural change in the plan. The shared shell touches every route; mistakes are visible everywhere.

### D7 — `app-deploy-7` · Brand sweep Mo → Tropo

- **What:** `.mb-*` / `.ma-*` / `.crew-*` CSS class renames to `.tropo-*` equivalents (most absorbed by primitives in D4-D5; sweep what remains). User-visible brand strings: `"Mo"` → `"Tropo"`, `"Mo Power"` → `"Tropo Power"`, `"Mo Author"` → `"Tropo Author"`. Storage keys: `mo-selected-kb` → `tropo-selected-kb` with read-fallback for one release. Assistant message labels. API route string surfaces.
- **Acceptance:** `grep -rE 'Mo\b|mb-|ma-|mo-|MindBridge' app/ lib/ components/` returns 0 matches. localStorage migration includes fallback path. typecheck clean · build clean · 4/4 routes 200.
- **Effort:** ~2.5h.

### D8 — `app-deploy-8` · Decompose `app/page.tsx` (822 lines)

- **What:** Split into focused sub-components co-located beside the page: `<QueryComposer>`, `<SSEController>`, `<ChatColumn>`, `<ArtifactColumn>`, `<ConfigColumn>`. No file over ~400 lines after.
- **Acceptance:** `app/page.tsx` ≤ 400 lines · each extracted sub-component has a clear prop type · SSE smoke-test (one query end-to-end) passes · typecheck clean · build clean · `/` route 200 + behaviorally identical.
- **Effort:** ~3h.

### D9 — `app-deploy-9` · Decompose `app/crew/page.tsx` (999 lines)

- **What:** Extract existing inline `StatusBadge`/`TierBadge`/`TaskStatusBadge` + add `<ThreadList>`, `<DashboardSummary>`, `<ProjectTree>` (or equivalent based on the page's actual structure). Replace remaining inline `style={{}}` hex codes with primitive callsites where possible.
- **Acceptance:** `app/crew/page.tsx` ≤ 400 lines · inline hex codes ≤ 20 · typecheck clean · build clean · `/crew` route 200 + behaviorally identical.
- **Effort:** ~3.5h.

### D10 — `app-deploy-10` · Decompose `app/projects/[id]/page.tsx` (989 lines)

- **What:** Split into sub-components (`<ProjectOverview>`, `<TaskList>`, `<BoardView>`, `<ProjectChat>`, etc. — final shape determined at implementation).
- **Acceptance:** `app/projects/[id]/page.tsx` ≤ 400 lines · typecheck clean · build clean · `/projects/[id]` route 200 + behaviorally identical.
- **Effort:** ~3.5h.

**Remaining effort estimate (D3 onwards):** ~25 hours of focused work, spread across 8 deploys. Calendar depends on cadence.

## Open Decisions

All five closed at lock (Mike sign-off 2026-05-16).

1. ✅ **D6 `<AppShell>` placement.** **LOCKED: `app/layout.tsx`** — Next.js idiom; removes the "remember to wrap" failure mode; route-group sub-layouts handle any future opt-out cleanly.
2. ✅ **D4-D5 variant library.** **LOCKED: `clsx` + `tailwind-merge`** — earn the abstraction. Drop-in upgrade to `cva` later if variant complexity demands it; not now.
3. ✅ **D5 author sub-app primitive adoption.** **DEFERRED to D5 step-0.** Decide after primitives are proven against the main app at D4. Plan covers both outcomes (primitivize or sibling-app).
4. ✅ **D7 brand sweep timing.** **LOCKED: engineering ships when ready.** Platform app isn't publicly deployed; no external coordination needed. A real launch-narrative moment is a separate decision later.
5. ✅ **D8-D10 sub-component naming.** **DEFERRED to each deploy's step-0.** Names emerge from actually opening each file; locking guesses now is premature.
