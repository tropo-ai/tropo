## §1 — Why this cycle exists

Three things compose the case for v1.19.0.

**First — the convergence direction needs concrete progress.** v1.17.0 declared Locks A-E. Lock A names the destination: everything that is a typed governed artifact converges to `vault/files/<uid>.md` with `member_of:` edges declaring context. v1.18.0 proved a substrate pattern — capsules can teach agents how to use each type when their bodies are structured cleanly. Pattern works; three capsules refactored as exemplars. v1.19.0 is the cycle named in Lock E to advance the convergence concretely on substrate that does not touch the boot path. Without v1.19.0 doing this work, the convergence direction is declared but not made real.

**Second — v1.20.0 needs to focus.** v1.20.0 is the boot-chain-reshape cycle: agent identity files migrate, boot resolution changes, the Tier 1/2/3 substrate gets re-architected. That is heavy work in its own right. Loading v1.20.0 with KB article migration and Studio-root playbook migration on top — both of which are easier than the boot-chain work — is the opposite of bite-sized. Doing those migrations at v1.19.0 frees v1.20.0 to do its named job cleanly.

**Third — the capsule library completion is hot.** v1.18.0 proved the pedagogy-first 5-section pattern on three capsules (avg 52% line reduction across playbook + activation-log + subsystem-hub). The pattern is fresh; the extraction-fidelity gate is established; the capsule-history.capsule typed primitive is authored and tested. Ten more capsules in the library carry the same body bloat the pattern resolves. Folding them into v1.19.0 means the library reads consistently for every future agent that loads a capsule.

The combined effect at ship: the convergence arc advances measurably on safe substrate, the capsule library reaches predictable shape, and v1.20.0 inherits a proven migration pattern with no new architectural debt.
