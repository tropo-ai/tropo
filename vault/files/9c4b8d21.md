## Purpose

The implementation step of build stage. Talos (or whichever engineering agent is running the pipeline) writes the code for the pages or features in the confirmed scope. Edits land at the tropo-ai/ repo root in `app/`, `components/`, `public/`, and supporting `lib/` as needed. The step closes when the changes compile in dev mode and render correctly at localhost:3000 for an eyeball check.

This is the only step in build that produces source code changes. verify-build (step-2) verifies what this step produced; it doesn't author new code.

## Structure

Leaf step. No children.

## Behavior

No confirmation gate. Executor enters the step; agent does the work; step closes when local dev-mode rendering is correct.

**Step work:**

1. Read the activation-root project's `pages_in_scope` to confirm what's being built.
2. Open or create the files needed. For pages: `app/<route>/page.tsx` and supporting components. For features: whatever the feature touches.
3. Run `npm run dev` (if not already running). Open localhost:3000 at the changed routes. Confirm the changes render correctly: layout, content, links, styles.
4. Author any per-deploy decisions or dependency notes as documents under the activation-root project (e.g., "Added Stripe dependency for pricing checkout — see follow-up at next-deploy").
5. Write `step_completed` event to `run.jsonl`. Stage advances to verify-build.

**On in-flight scope expansion:** if the implementation surfaces a need for scope beyond what was confirmed at step-0, pause and ask Mike before expanding. Do not silently grow the scope.

## Inputs

- Activation-root project (from step-0): `pages_in_scope`, `theme`, deploy number.
- Current state of the repo at the deploying branch.

## Outputs

- Source code changes at `app/`, `components/`, `public/`, `lib/` as needed.
- Optional per-deploy documents under the activation-root project (decisions, dependency notes).
- `run.jsonl` events: `step_started`, `step_completed`.
