*Voice-review step. Orpheus reads each touched doc against the voice canon and confirms it lands.*

This step is what makes "pristine documentation" a substrate-checkable claim rather than an aspirational one. The voice metric for each doc is per [voice-review.skill.md](../../.tropo/skills/voice-review.skill.md) (Phase C companion deliverable), which carries the intent + goal — voice consistency at a stranger-encounter-passing standard — and operationalizes the three layers (tone consistency + lore alignment + stranger-encounter test) as guidelines. The human walking the review exercises judgment per piece: weight the layers, skip when not load-bearing, add a fourth lens when the piece needs one. The step gate enforces that the skill was run (substrate evidence exists) and the human signed off, not a hard-coded PASS pattern on the three layers.

**What it does:**
1. Enumerates touched docs from Step 2 by tier
2. For each tier:summary + tier:subsystem doc (`voice_review_required: true` default): runs full voice review per voice-review.skill.md three layers
3. For each tier:spec doc: runs self-spot-check on prose-heavy sections (§Intent, §Why, §Studio Shop Signage, §How to Validate body prose) per doc-spec.capsule §Tier 3 procedural note; not gating but inline-recorded
4. Authors voice-review-notes entries at vault/files/<uid>.md (type: note, subtype: voice-review-notes) capturing per-doc review findings; one entry per fully-reviewed doc
5. Collects voice-review-notes UIDs for Step 5 acceptance_evidence

**Trust level human-walked rationale:** voice review is Orpheus judgment; no machine substitute. The substrate (voice-review-notes entries) is the receipt of the judgment; the judgment itself can't be auto-retried on failure (an agent re-running voice review against an unchanged doc produces the same result; the failure mode is the doc, not the review).

**Failure modes:**
- tier:summary or tier:subsystem doc fails voice review → step does NOT close; Orpheus returns to Step 2 to amend the doc; voice-review-notes entry records the FAIL + what needs to change
- tier:spec doc spot-check surfaces drift → Orpheus amends inline if trivial; surfaces to Step 2 if substantive
- voice-review.skill.md unavailable → activation fails with explicit "Phase C skill dependency missing" error

**Hands off to:** step 4 validate-cross-references, when all required voice-review-notes entries authored + reviewed FAILs resolved.
