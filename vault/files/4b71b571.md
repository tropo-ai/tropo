## Purpose

The deploy stage of app-pipeline. Takes the verified production artifact from build, commits and pushes it to the platform remote (`tropo-ai/tropo-ai`) — which triggers the host's auto-deploy if wired — then confirms the deployed instance renders the changes correctly. Two steps split the work: **commit-and-push** is the irreversible git action with Mike-confirmation gate; **smoke-test** is the post-deploy verification.

This is the terminal stage of the pipeline. On smoke-test close, the pipeline-run completes and the activation-root project flips to `state: done`.

**Does NOT govern:** scope decisions (specify); implementation or production-build verification (build); rollback on failure (not declared at v1.0.0 — see Known Enforcement Gaps).

## Flow Rules

Linear. step-3 (commit-and-push) → step-4 (smoke-test). step-3 declares `requires_confirmation: true` — pipeline-run pauses on entry until Mike approves the commit message and authorizes the push. Once confirmed, executor commits + pushes; host's auto-deploy fires if wired.

step-4 verifies the deployed instance — public live URL if declared on the activation-root project, otherwise local production-preview. On step-4 close: deploy stage closes, pipeline-run closes, activation-root project transitions `state: standing → state: done`.

The push-approval gate at step-3 encodes Talos's hard rule: never push to remote without Mike's approval. The pipeline encodes that gate structurally so it doesn't depend on agent memory.

## Cold-Boot Walk-Through

Build has cleared with `app-deploy-1` artifact verified locally. Pipeline-run enters deploy stage. Step-3 (commit-and-push) opens. Executor authors a structured commit message naming the deploy number, theme, files-touched, and files-changed summary. Per `requires_confirmation: true`, the run pauses with a `pause_started` event in run.jsonl containing the proposed commit message. Mike reviews, redirects if needed, and confirms with a `pause_resumed` event. Run resumes; executor runs `git add <files-in-scope>`, `git commit -m "<message>"`, and `git push origin main`. If a public deploy target is wired, the host's auto-deploy fires; otherwise the push lands at remote and smoke-test verifies the local production-preview. step-3 closes; flow advances to step-4. Step-4 (smoke-test) opens. Executor verifies per the activation-root project's `live_url:` declaration (if present) or against local preview (if absent). Reports the verification surface to Mike for eyeball check. On success: step-4 closes, deploy stage closes, `run_complete` event written, activation-root project (`app-deploy-1`) flips `state: standing → state: done`, pipeline-run flips to `status: complete`.
