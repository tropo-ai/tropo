## Purpose

The runtime-first-step of dev-pipeline. Transitions the work-item from `accepted` to `active` by writing an acceptance record and updating the work-item's status field. This step IS the implementation of the inbox-to-active transition described in the vault inbox primitive (e1c47a9f Part A).

Runs before any specify work begins. Every dev-pipeline run starts here.

## Structure

Leaf step. No children.

## Nodes

No child nodes. This is a leaf WorkflowNode.

## Flow Rules

Completes when: the work-item's `accepted_by:` array has a new entry with `accepted_by_uid: <this-run-uid>` and `date_accepted: <today>`, and the work-item's `status:` is `active`.

Advances to: generate-release-plan (180e9108).

## Cold-Boot Walk-Through

The pipeline-run executor reads the work-item referenced by this run. If `status: accepted`, it writes the acceptance record and sets `status: active`. If status is already `active` (rare — re-run scenario), it logs and proceeds without writing a duplicate record. The step is done.
