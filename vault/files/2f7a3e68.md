## Purpose

The runtime-first-step of web-pipeline. Accepts the human-issued deploy trigger ("run web-pipeline as web-deploy-N, theme: '<one-line>', scope: <pages-or-features>"). Authors the activation-root project per pipeline.capsule Rule 10. Pauses for Mike to confirm the deploy number + theme + scope before flow advances to build.

This is the scope-confirmation pause point of the pipeline. Build and verify-build run through without pauses once specify clears; deploy has its own push-confirmation gate at step-3.

## Structure

Leaf step. No children.

## Behavior

**`requires_confirmation: true`.** When pipeline-run executor enters this step, it transitions to `status: paused` BEFORE step work begins, writes a `pause_started` event to `run.jsonl` with:

- `data.reason: "human-confirmation-required"`
- `data.proposed_deploy: <web-deploy-N>`
- `data.proposed_theme: <one-line-theme>`
- `data.pages_in_scope: [<list of page/feature paths>]`

The run waits. Mike (the authorized confirmer per pipeline-run.capsule v1.1+) appends a `pause_resumed` event with `data.confirmation_granted_by: mike` and any redirects on number, theme, or scope. Run resumes; step work executes.

**Step work (once confirmed):**

1. Author the activation-root project for the deploy at `vault/files/<deploy-project-uid>.md`:
   ```yaml
   type: project
   name: "web-deploy-N"   # the Mike-confirmed number
   title: "web-deploy-N — <theme>"
   status: active
   state: standing   # transitions to done at end of pipeline-run
   owner: talos
   member_of: ["7b2e94c1"]   # web-pipeline root
   pages_in_scope: [...]
   theme: "<one-line>"
   ```
2. Write `step_completed` event to `run.jsonl`. Stage advances to build.

## Inputs

- `deploy_version` (string, required) — e.g., `web-deploy-3`. Mike-confirmed.
- `deploy_theme` (string, recommended) — one-line description of what this deploy is about. Mike-confirmed.
- `pages_in_scope` (array of strings, required) — paths or feature names included in this deploy. Mike-confirmed.

## Outputs

- Activation-root project at `vault/files/<deploy-project-uid>.md` — becomes the graph parent for all downstream artifacts the run produces.
- `run.jsonl` events: `pause_started`, `pause_resumed`, `step_started`, `step_completed`.
