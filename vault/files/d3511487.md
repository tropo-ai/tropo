*Substrate authoring step. Vela primary; Talos composes when machine_executable_script requires non-trivial engineering (e.g., harness extension or new runner). Argus reviews substrate-coherence as separate-context verifier per pipeline.capsule v3.0 verification-class pattern.*

**Why this step's owner is split:** most test substrate is authored by Vela (validator extensions, deterministic assertion DSL expressions, agentic_review commissioning records, manual_walk prompts). Machine-executable scripts requiring meaningful engineering implementation (e.g., wiring a new test runner, instrumenting a code path for assertion capture) compose with Talos T9 per the engineering-lane pattern. Argus reviews substrate-coherence at separate context to catch box-checking misses Vela's authoring context can't see.

**What it does:**
1. For each plan section from step 1, author the test substrate per verification_method:
   - machine_executable_script → write .py/.sh/.ts file at named path; ensure exits non-zero on behavior break; pair-call Talos if engineering complexity warrants
   - structural_check → author validator extension function in tropo-validate.py; register in main() dispatcher
   - deterministic_assertion → encode DSL expression in plan substrate; engine evaluates at step 3
   - agentic_review → prep commissioning record file at agents/sa/<sa.class>/activation-log/NNN-record.md; populate [PENDING] block; dispatch fires at step 3
   - manual_walk → encode walk prompt in plan substrate; Mike walks at step 3 if dispatched
2. If harness_framework_changes_required flagged at step 1: cycle scope EXPANDED — Vela authors harness extension substrate; Talos engineers; Argus reviews capsule/spec amendments. Step 2 does NOT close until harness extension landed + working.
3. Argus reviews substrate-coherence via argus-vela channel (separate context per verification-class pattern); catches anti-box-checking misses

**Anti-box-checking discipline:** if you find yourself writing a "test" that doesn't have a meaningful pass/fail, the substrate is wrong — go back to step 1 and refine the verification_method choice. The capsule's anti-box-checking gate enforces this at step 4; catching it at authoring time is cheaper.

**Hands off to:** step 3 run-test-substrate.
