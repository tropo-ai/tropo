## Purpose

The production-build gate of build stage. Runs `npm run build` (full Next.js production build with typecheck and route generation) against the changes from step-1, plus `npx tsc --noEmit` for strict type safety, then starts the production preview and confirms the affected routes render correctly. Catches a real class of bugs that dev mode hides — missing image imports, route conflicts, type errors in unused branches, environment-variable misses, Tailwind purge surprises.

This step exists as a separate gate (not collapsed into step-1) specifically so it can't be hand-waved. If the production build fails or type-check fails, the deploy doesn't ship.

## Structure

Leaf step. No children. Terminal in build stage; on close, flow advances to deploy.

## Behavior

No confirmation gate. Executor runs the production build; step closes only on green build + clean preview.

**Step work:**

1. Run `npx tsc --noEmit` from the `tropo-ai/tropo-ai` repo root. Captures type errors not caught at dev-time.
2. Run `npm run build` from the repo root.
3. **On build or typecheck failure:** capture the error to `run.jsonl` (`step_paused` event with `data.reason: production-build-failure | typecheck-failure`, `data.error: <output>`). Step pauses. Agent fixes the error in step-1 territory (the implementation files), re-runs. Repeat until green.
4. **On green build + typecheck:** start the production preview: `npm start` (or `npm run preview` per package.json). For each affected route in the activation-root project's `changes_in_scope` (or, when scope is non-route — e.g., a lib refactor — pick representative routes that exercise the changed code): curl the production preview at `localhost:3000/<path>`, confirm HTTP 200, confirm key content strings render (one or two stable strings per route — page title, primary heading, or known content marker).
5. **For `expected_visual_diff: zero` refactors:** also do an eyeball baseline comparison against the pre-change render. Visual-diff tooling (Playwright, percy.io) is the v1.1+ path; v1.0 is manual eyeball.
6. **On preview failure (route 404, content missing, render error, visual diff for zero-diff refactor):** capture to `run.jsonl`, pause, agent fixes, re-run build.
7. **On all-green:** write `step_completed` event to `run.jsonl`. Stage advances to deploy.

**Scope:** the routes in (or exercised by) `changes_in_scope` are checked here. The full app doesn't get re-tested per deploy — that's deploy stage's smoke-test concern, and even there it's scoped to changed surfaces.

## Inputs

- Source code from step-1.
- Activation-root project: `changes_in_scope`, `expected_visual_diff`.

## Outputs

- Confirmation that `npx tsc --noEmit` succeeded.
- Confirmation that `npm run build` succeeded.
- Confirmation that the production preview renders the affected routes correctly.
- `run.jsonl` events: `step_started`, `step_paused` (on failures), `step_completed`.

| Gap | What closes it | Target release | Owner |
