# What is Tropo?

**Relations**

| Relation | Target |
|---|---|
| Governed by | [kb-article (4cb20382)](4cb20382.md) |

Tropo is an operating system for human-AI teams. It turns a shared folder into a governed multi-agent system — no code, no server, no database.

## The One-Liner

Describe how you work. The system captures it. Now your expertise runs without you.

## Who It's For

Non-technical founders, consultants, team leads, and career technologists who want to build with AI agents but don't write code. If you can describe how your team works, you can build agents in Tropo.

## How It Works

A Tropo vault is a folder on your computer containing markdown files. Each file serves a purpose:

- **Agent files** define your AI teammates — their roles, values, scope, and instructions
- **The operating agreement** defines who has authority and what boundaries exist
- **Playbooks** capture how processes should run — structured guides any agent can follow
- **Channels** record communication and activity
- **The registry** tracks everything in the vault

You open your AI tool, point it at the vault, and your agents boot with full context — they know who they are, what they can do, and how you work.

## Why Files?

Every other multi-agent system requires infrastructure — databases, APIs, orchestration code. Tropo uses files because:

1. **Any AI can read them.** Claude, GPT, Gemini — they all read markdown.
2. **Any human can read them.** Your agent's rules are in plain text, not hidden in a database.
3. **Git can version them.** Every change is tracked automatically.
4. **You own them.** The vault lives on your machine. No vendor lock-in.
5. **They're portable.** Move the folder anywhere and it still works.

## The Three Things You Need to Know

1. **Folders have rules.** Each folder has an AGENTS.md file that tells agents what they can and can't do there.
2. **Files have identity.** Every file is tracked with a unique ID so nothing gets lost.
3. **The system is governed.** Your operating agreement defines authority. Agents propose, you decide.

Everything else — the technical details, the registry format, the charter schema — your agents handle for you.
