## Purpose

The deploy stage of web-pipeline. Takes the verified production artifact from build, commits and pushes it to remote (which triggers the host's auto-deploy), then confirms the live site at tropo-ai.com renders the changes correctly. Two steps split the work: **commit-and-push** is the irreversible git action with Mike-confirmation gate; **smoke-test-live** is the post-deploy verification against the live URL.

This is the terminal stage of the pipeline. On smoke-test-live close, the pipeline-run completes and the activation-root project flips to `state: done`.

**Does NOT govern:** scope decisions (specify); implementation or production-build verification (build); rollback on failure (not declared at v1.0.0 — see Known Enforcement Gaps).

## Flow Rules

Linear. step-3 (commit-and-push) → step-4 (smoke-test-live). step-3 declares `requires_confirmation: true` — pipeline-run pauses on entry until Mike approves the commit message and authorizes the push. Once confirmed, executor commits + pushes; host's auto-deploy fires.

step-4 waits for the host to report deploy complete, then verifies the live site. On step-4 close: deploy stage closes, pipeline-run closes, activation-root project transitions `state: standing → state: done`.

The push-approval gate at step-3 encodes Talos's hard rule: never push to remote without Mike's approval. The pipeline encodes that gate structurally so it doesn't depend on agent memory.

## Cold-Boot Walk-Through

Build has cleared with `web-deploy-3` artifact verified locally. Pipeline-run enters deploy stage. Step-3 (commit-and-push) opens. Executor authors a structured commit message naming the deploy number, theme, pages-touched, and files-changed summary. Per `requires_confirmation: true`, the run pauses with a `pause_started` event in run.jsonl containing the proposed commit message. Mike reviews, redirects if needed, and confirms with a `pause_resumed` event. Run resumes; executor runs `git commit` and `git push origin <branch>`. Host's auto-deploy fires from the push. step-3 closes; flow advances to step-4. Step-4 (smoke-test-live) opens. Executor waits for the host's deploy-complete signal (mechanism host-specific — see Known Enforcement Gaps). Once live, executor curls the changed pages at `https://tropo-ai.com/<path>`, confirms HTTP 200 + key content strings, and reports the live URLs to Mike for eyeball check. On success: step-4 closes, deploy stage closes, `run_complete` event written, activation-root project (`web-deploy-3`) flips `state: standing → state: done`, pipeline-run flips to `status: complete`.
