*Five questions from operators trying Tropo for the first time, with plain answers. If you're stuck, ask your AI — it can read this file and walk you through any of the answers in more detail.*

---

## 1. "I created my first agent. Now what?"

Your agent is ready to work. **Want me to walk you through launching it right now? Just say so — I'll guide you step-by-step.**

If you'd rather do it yourself, here's what to do:

1. **Launch the agent in a new session.** In your AI tool (Claude Code, Cursor, etc.), open a fresh session pointed at this folder. Then ask the AI to read your agent's activation file — the playbook just told you the path (it'll be `agents/<your-agent-name>/<your-agent-name>-activation.md`). The AI reads the activation file and becomes that agent for the session.
2. **Tell it the first thing you want.** The agent has its mission and values loaded — it knows what it's for. Talk to it the way you'd talk to a colleague who's new on day one. Plain English. No special syntax.
3. **Watch what it does.** It authors files in `vault/files/` and updates `channels/ops.md` as it works. Open those files anytime — they're plain markdown.

If you set up a project alongside the agent, the project's board lives at `vault/files/<project-uid>.md`. Your agent will tell you the project's UID in conversation.

---

## 2. "Why is there so much vocabulary?"

Honest answer: there is. Three vocabulary tiers — you only need the first:

- **Words you'll actually use:** project, task, decision, agent, channel. These describe what you and your agents are doing day-to-day.
- **Words the AI uses to keep things organized:** vault, capsule, registry. These are how the AI keeps your work findable across sessions. You'll hear them but you don't need to author them.
- **Words you can skip:** UID, frontmatter, schema, validator, ADR. These are how the AI executes its own work. You don't need to learn them to use Tropo.

If a word feels confusing, ask your AI: *"what does X mean in Tropo?"* It can read the Tropo knowledge base and explain in your context.

---

## 3. "Can I just tell my agent what to do?"

Yes. Tropo agents work like colleagues — describe what you want in your own words; they figure out the structure.

You DON'T need to:
- Write tasks in a specific format
- Use any special syntax or markup
- Know what type of artifact (task, decision, project, etc.) to create
- Memorize file paths

You DO need to:
- Tell your agent what you want done, in your own words
- Approve what they propose before they write files — they'll always ask first
- Keep `channels/ops.md` open if you want to see what they did

If your agent gets confused or asks for clarification, give them more context. They learn what matters to you across the session.

---

## 4. "How do I know my agent actually did anything?"

Two places to look:

- **`channels/ops.md`** — your agent logs every meaningful action here in plain English. Open it in any text editor or your AI tool. You'll see one-line entries: who did what, to which file, when.
- **`vault/files/<uid>.md`** — every file your agent creates lives at one of these paths. Your agent will tell you the UID in conversation (e.g., *"I wrote up the decision at `vault/files/a4f8c291.md`"*). Click the link or open the file directly.

If your AI tool supports markdown preview (most do), both files render cleanly with the metadata hidden.

If you want a high-level view: ask your AI *"what did we work on today?"* — agents have memory and can summarize the session.

---

## 5. "Where do I find what I created?"

Three navigation paths, easiest first:

1. **Ask your agent.** Say *"what did we make this week?"* or *"show me the project we started yesterday."* Agents have memory and read the vault index — they'll list what's there.
2. **Open the project board.** If you started a project, its board lives at `vault/files/<your-project-uid>.md`. The board shows tasks, status, and recent activity.
3. **Browse the vault.** Every file you and your agents have created lives at `vault/files/<uid>.md` (eight-character hex names like `a4f8c291.md`). The frontmatter `title:` field tells you what each file is. If your AI tool offers a file tree, browse `vault/files/` directly.

For finding something specific by keyword, your AI can grep `vault/00-index.jsonl` — that's the master index of every governed file in your Studio.
