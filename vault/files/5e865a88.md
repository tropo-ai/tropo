## Purpose

The build stage of app-pipeline. Takes the confirmed scope from specify and produces a deployable artifact verified locally. Two steps split the work: **implement** writes the code and confirms dev-mode rendering; **verify-build** runs the production build (full typecheck + route generation) and confirms the production preview renders the changes correctly.

The split matters because dev mode hides a real class of bugs that `npm run build` catches — missing image imports, route conflicts, type errors in unused branches, environment-variable misses, Tailwind purge surprises. Making verify-build a separate step means the production-build gate can't be hand-waved.

**Does NOT govern:** scope decisions (specify); the commit, push, or post-deploy verification (deploy).

## Flow Rules

Linear. step-1 (implement) → step-2 (verify-build). No confirmation gates in this stage. On step-2 build failure or preview-render failure: step pauses; agent fixes and re-runs the build until green. On green build + clean preview: stage closes; flow advances to deploy.

## Cold-Boot Walk-Through

Specify has cleared. Pipeline-run enters build stage with deploy `app-deploy-1` and scope `[app/styles/tropo-tokens.css, app/globals.css]` confirmed. Step-1 (implement) opens. Talos edits the files in scope, runs `npm run dev`, confirms the dev preview renders identically to the pre-change baseline. Notes any decisions or per-deploy details as documents under the activation-root project. Writes `step_completed` event; flow advances to step-2. Step-2 (verify-build) opens. Executor runs `npm run build` from the repo root. Full production build succeeds (or pauses on failure for Talos to fix). On green: executor starts `npm start`, curls the affected routes at the production preview (localhost:3000), confirms HTTP 200 + key content strings render. For refactor-class changes (zero visual diff expected), the check is "produces the same content as before." For feature-class changes, the check is "renders the new content." Step-2 closes; build stage's next_steps fires; flow advances to deploy.
