## Intent

One-time v1.1 index migration. Populates the four v1.1-introduced fields (scope, project, parent, created_by) by reading frontmatter from each vault file. Idempotent — re-running overwrites with current state. **Status: deprecated** — v1.1 migration completed; rebuild-vault.py handles all routine index work.

## Invocation Protocol

```bash
python3 .tropo/scripts/populate-index-v1.1.py            # apply
python3 .tropo/scripts/populate-index-v1.1.py --dry-run  # preview
```

## Input / Output

**Input:** `--dry-run` flag (optional). **Output:** updated `vault/00-index.jsonl` (or stdout in dry-run mode).

## Governance

**Permission scope:** writes `vault/00-index.jsonl`. **Audit:** not required (one-time migration). **Deprecation:** retained for historical-recovery scenarios; new index work uses rebuild-vault.py.

## Verification

Exit code 0 means PASS. Spot-check a sample of index entries to confirm the four v1.1 fields are populated.

## Failure Modes

| Exit | Meaning | Response |
|---|---|---|
| 0 | Migration applied | Verify spot-check |
| 1 | File parse failure | Read error; fix malformed frontmatter; rerun |
