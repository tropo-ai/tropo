## Purpose

The author stage produces the finalized primary-surface draft. It accepts the outline from specify, drafts the full primary-surface content, optionally routes to a designated reviewer, and locks the draft for release. Author ends when the draft is publish-ready.

Does NOT govern: the multi-surface release work (steps 6-9 in release stage), or any content-structure rules for the artifact itself (those live in future per-surface capsules — `pub-substack.capsule`, etc.).

## Structure

```
author (a1d8f273)
├── step-3: draft (5c1e8f43)
├── step-4: review (b6d3a912) — OPTIONAL pause point
└── step-5: finalize (2e9c4f51)
```

Flow: step-3 → step-4 → step-5 (linear). step-4 is OPTIONAL — pipeline-runs advance through it without ceremony when no reviewer is designated on the activation root.

## Flow Rules

Linear, forward-only. step-4 (review) is OPTIONAL — when the activation root's `reviewer:` field is null/absent, the executor advances through step-4 immediately and proceeds to step-5. When a reviewer is designated, the executor waits for an external "review complete" signal before advancing. step-5 (finalize) is terminal in this stage; author advances to release when its three steps complete.

## Cold-Boot Walk-Through

The pipeline-run enters author stage with the outline already committed. step-3 (draft) produces the full primary-surface draft as a `type: document` artifact, member of the activation root. step-4 (review) closes immediately if no reviewer is designated, or waits for reviewer signoff if one is. step-5 (finalize) locks the draft. When step-5 closes, author advances to release.

## Known Enforcement Gaps

step-4 optional-step semantics are convention-only at v1.0.0. See root pipeline ([e2f4a8c1](e2f4a8c1.md) §Known Enforcement Gaps) for full list and target releases.
