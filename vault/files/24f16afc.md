## Purpose

For each arch-spec authored in step-2, produces three required artifacts: (a) a research node documenting relevant context and prior art, (b) one or more sprint plans each with a testing plan, (c) a documentation node updating the master tropo-subsystems documentation system.

This is the core execution step of the build stage. It is the only step in build at v1.0.0. Step is not done until ALL arch-specs declared in the release-plan have their complete artifact sets.

## Structure

Leaf step. No children.

## Nodes

No child nodes. This is a leaf WorkflowNode.

## Flow Rules

Completes when: for every arch-spec in the release-plan, all three artifact types exist (research node, sprint plan(s) with testing plans, docs node). No arch-spec may be artifact-incomplete when this step closes.

**(v1.0.1 update)** No longer terminal within build stage. On completion, advances to step `update-subsystem-canonical-docs` (9d4f7e21) — the v1.9.2 NEW step that auto-fires Rule 12 hub release_history derivation. Build stage advances to deploy (3a7dbdda) only after both steps complete.

The step is effectively a loop over N arch-specs. At v1.0.0 this loop is hand-executed by the agent. See Known Enforcement Gaps.

## Cold-Boot Walk-Through

The executor reads the release-plan's arch-spec list. For each arch-spec (processed sequentially or in parallel depending on agent capacity):

1. **Research node** — Author a research entry capturing context, prior art, relevant ADRs, and open questions relevant to this sub-system.
2. **Sprint plan(s)** — Author one or more sprint plans. Each sprint plan includes: scope, tasks, definition-of-done, and a testing plan. *(sprint-plan capsule type is a Known Enforcement Gap.)*
3. **Docs node** — Author a documentation node that will update the master tropo-subsystems documentation on deploy. The node captures what changed in this sub-system and why.

When all three artifacts exist for all arch-specs, the step closes. Build stage done. Deploy begins.
