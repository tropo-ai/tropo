## §What This Subsystem Is

The Tropo Test Harness is the substrate that answers four distinct release-quality questions:

1. **Is the substrate well-formed?** (form-check; cycle-specific; Argus-authored vela-test-plans)
2. **Can a stranger encounter the Studio and get to their first agent?** (user-experience first-encounter; cycle-comparable; persona walks)
3. **Can a user exercise Tropo Work primitives at depth?** (user-experience usage-pattern; cycle-comparable; persona walks with scenario tails)
4. **Do specific features still work mechanically?** (regression verification; persistent per-feature; feature-specific test cases)

Each primitive runs as a **release-step gate**. Together they form a complete quality envelope: form-check + first-touch UX + depth-of-use UX + per-feature regression. No single primitive is sufficient; together they are.

The harness is named at this hub; the components live at their canonical type-locations (playbooks live where playbooks live; personas live where personas live; test cases will live in their own kernel folder). This hub is the **discovery surface** + the **doctrine surface** — what the harness IS, why each component composes the way it does, and where each piece lives.

## §Why This Is First-Class (Mike-V47 Calibration 2026-05-18)

V47's initial design-walk pulled back from subsystem-class elevation per a strict reading of "earn the abstraction": *"Release test harness is a TOOLING surface that serves Tropo Governance, not a peer subsystem."* Mike overrode that read.

**The calibration:** when substrate is load-bearing AND compounds across every release AND has multi-component composability, elevation to subsystem-class is the correct posture — not over-investment. The test harness clears all three:

- **Load-bearing:** the harness IS the release ship-gate. Without it, releases ship on substrate-verification alone (vela-test-plan) — which v1.41 + v1.42 dogfoods empirically proved is insufficient (cold-boot walk caught two P0 Stream-E regressions vela-test-plan missed).
- **Compounds across releases:** cycle-comparable signal accumulates. The harness gets MORE valuable across releases, not less. Each cycle's baseline informs the next.
- **Multi-component:** four primitives at minimum (vela-test-plan + cold-boot-walk + tropo-work-walk + test-cases); each at its canonical type-location; needs a coherent discovery surface.

That clears the substrate-subsystem-class bar. Tropo Work earned it; Tropo Governance earned it; the Tropo Test Harness earns it.

## §Rubric + Scoring Discipline (cold-boot-walk + future tropo-work-walk)

Per `release-cold-boot-walk.playbook v1.1`:

- **3 binary gates per persona** — Greeted clean / First agent created / Create-then-use closed
- **1 holistic 1-10 score** with one-paragraph persona-lens rationale
- **Gap inventory** per the sa.cold-boot naive-session discipline (every off-target read, every ambiguity, every friction point logged)
- **Aggregate verdict:** SHIP-CLEAR (all gates pass + avg ≥ 8.0) / MIXED (gates pass + avg 6.0-7.9) / SHIP-BLOCKED (any gate fails OR avg < 6.0 OR cycle-blocking findings)
- **Cycle-comparable signal:** same harness + same personas + different substrate cycle = score-delta + gap-class-delta inform next-cycle scope priorities

Personas: three production (engineer / operator / enterprise); Studios may amend or add. Cross-sleeve coverage (Opus + Sonnet + Haiku) is a future variant (`target_sleeve:` field in persona substrate).
