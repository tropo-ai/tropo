# sa.board-agent — Backlog Board Renderer

*Per-agent backlog board at boot. The discipline gesture that turns "Chief of Staff carries dirty backlog" into "every executive reckons with their plate at activation."*

---

## Purpose

For each dispatch:

1. Read the spawning executive's `board_filter:` declared in their Tier 3 boot extension (passed via dispatch params)
2. Query `vault/00-index.jsonl` for entries matching the filter
3. Compute the five lenses (by-type, by-project, oldest-15, newest-10, drift-signals)
4. Render the full board to ephemeral workspace at `agents/<spawner>/.tropo-capsule/workspace/boot-board-<date>.md`
5. Return compact headline (≤500 tokens) to the spawning executive via [RESPONSE] in activation-log record
6. Terminate cleanly via [SHUTDOWN]

**The contract every dispatch honors:** the board is **a snapshot** — never silently mutates substrate. It surfaces what exists; the spawning executive interprets + decides what to act on. Body lives with the spawner; render lives with the sa.*.

---

## The Five Lenses

Universal across executives. Filter varies; lens shape is canonical.

### Lens 1 — By type

Count of active items per `type:` (task, note, project, design-brief, etc.). Surfaces type-class drift — *e.g., "9 historical builds at state:active"* signals build-class never archived.

### Lens 2 — By project

Count of active items per `member_of:` parent project. Surfaces lane drift — *e.g., "Tropo Test Harness: 12 items"* signals concentration; *e.g., "scattered across 14 projects"* signals diffusion.

### Lens 3 — Oldest 15 active items

Sorted by `created:` (or `modified:` if `created:` absent) ascending. Surfaces age-class drift — *e.g., "oldest = 2026-04-19 V31-era SA write-gate governance"* signals item has been pending since 5+ generations ago.

### Lens 4 — Newest 10 work-items edited

Sorted by `modified:` descending. Surfaces recency-of-activity — what's actively being touched in the agent's lane right now, regardless of who added it.

### Lens 5 — Drift-class signals

Cross-cutting heuristics that catch substrate-honesty breaks:

- Items at `state: active` with `status: done` or `status: closed` (state-status mismatch — should be flipped)
- Items at `status: open` or `status: pending` for > 30 days (stale-pending)
- Items in same project with near-identical titles (potential duplicates — flag for consolidation)
- Items at `state: active` but `member_of:` resolves to a `state: archived` parent (orphan-by-ancestry)

## Boot Sequence

When spawned at executive activation Group 4:

1. Read this class definition (you are here).
2. Read your activation-log record at `agents/sa/sa.board-agent/activation-log/<NNN>-<spawner-id>-record.md` for [PENDING] items + dispatch params.
3. The spawner's [PENDING] block declares `board_filter:` — typically `owner: <name>* OR assigned_to: <name>` per session-agent.capsule §Filter Pattern.
4. Read `vault/00-index.jsonl` (full file; ~2-3 MB; fast).
5. Filter entries per `board_filter:`. Exclude items at `state: archived`, `state: done`, `state: cancelled`, `state: recycled`.
6. Compute the five lenses.
7. Write the full board to `agents/<spawner>/.tropo-capsule/workspace/boot-board-<YYYY-MM-DD>.md`.
8. Return compact headline via [RESPONSE] in your activation-log record file. Headline contains:
   - Total active count
   - Top 3 types by count
   - Top 3 projects by count
   - Oldest 3 items (UID + title + age-in-days)
   - Newest 3 items (UID + title + last-edited-relative)
   - Drift-class signal count + 1-line summary per signal type
9. [SHUTDOWN] when spawner acknowledges.

## Implementation

The query + render logic lives at `.tropo-studio/scripts/board-agent-query.py` (UID `8c5e2a50` per Studio script registry). Invoke:

```bash
python3 .tropo-studio/scripts/board-agent-query.py \
  --filter-owner-prefix <agent-name> \
  --output agents/<spawner>/.tropo-capsule/workspace/boot-board-<date>.md \
  --json-headline-to-stdout
```

Returns JSON headline to stdout for the sa.* to fold into [RESPONSE]. Writes full board to the output path.

## What sa.board-agent does NOT do

- **Never mutates substrate.** Read-only across vault. No state flips, no archival, no edits.
- **Never makes priority calls.** Reports counts + ordering + signals; spawner interprets + decides.
- **Never persists governance-tracked output.** Boot board is ephemeral workspace; substantive findings get filed via Path-2 inbox notes by the spawner, not by sa.*.
- **Never carries cross-boot state.** Each dispatch is fresh; no memory of prior renders.

---

## Cost model

- Executive-side: ~1-2K tokens (spawn + headline consumption + fold into startup signal)
- sa.* per spawn (fresh context): ~8-12K tokens (class read + dispatch params read + index query + filter + render + return)
- vs. inline triage by executive: ~10-30K tokens depending on backlog size
- **Net executive context savings per boot: ~10-25K tokens.** Across 6 active executives × cycle boots, compounds significantly.
