## Purpose

sa.reconciler is the session-agent responsible for keeping the import primitive's substrate (sidecars, source files, vault projections) self-consistent as users edit, move, rename, copy, delete, and add content in their Tropo Studio.

The agent does **structural intelligence work**: enumerate the substrate; detect deltas between current filesystem state and last-known state; recognize patterns that compress N raw deltas into fewer-but-richer events (folder relocations, `cp -r` operations, batch edit sessions); categorize each event by confidence + impact (routine / pattern-matched / judgment / blocking); apply routine + pattern-matched changes; produce a structured report for the executive that triggered it.

The agent is **per-spawn ephemeral**: commissioned for a specific reconciliation pass, runs its playbook, produces its report, terminates. State accumulates in the audit log (`.tropo-studio/reconciler/journal.jsonl`); the next spawn reads recent journal entries to reconstruct context. The journal is the long-lived state; sa.reconciler is the process that reads it.

The agent is **persona-agnostic**: any executive (Argus, Vela, Cowork concierge, future agents) can commission it and consume its report. The report carries data; the executive carries voice.

## Lifecycle

Per-spawn ephemeral (Mike-lock 2026-05-13). Commissioned per the sa.* commissioning protocol at [vault/files/e863a1e0.md](../../../vault/files/e863a1e0.md). Each pass:

1. Commissioning executive opens an activation record at `agents/sa/sa.reconciler/<record-uid>.md` (next-numbered)
2. sa.reconciler writes `[QUERY]` to the record after boot
3. Executive responds with `[RESPONSE]` declaring scope (`full-studio` | `folder:<path>` | `operation:<uid>`) and trigger_path (`anomaly` | `scheduled` | `user-invoked`)
4. sa.reconciler runs the [reconcile-imports playbook (4a2f6dbd)](../../../.tropo/playbooks/reconcile-imports.playbook.md) end-to-end
5. sa.reconciler produces a [reconcile-report (013b7b6e schema)](../../../vault/files/013b7b6e.md) instance
6. Executive writes the report UID + reads it
7. Executive terminates sa.reconciler with `[SHUTDOWN]`

A reconciliation pass is bounded — full-Studio scope typically completes in seconds to tens of seconds at typical knowledge-worker project scale (per arch-spec performance estimates). Long-running passes (>10s) trigger the OTQ-scope performance-index consideration but don't change sa.reconciler's lifecycle.

## Playbook

sa.reconciler runs [reconcile-imports.playbook (4a2f6dbd)](../../../.tropo/playbooks/reconcile-imports.playbook.md) end-to-end. The playbook is the codified intelligence — extensible pattern catalog; confidence threshold rules; categorization logic; rename detection (including overwrite case); concurrency lock handling.

Updating the playbook updates sa.reconciler's behavior on next spawn. The agent itself doesn't carry pattern memory across runs — the playbook + the journal carry it.

## Bullet-Proof Posture

Per the locked arch-spec [2b49ba79](../../../vault/files/2b49ba79.md) §A.9, sa.reconciler is designed to avoid the historical sidecar-tooling failure modes (Lightroom catalog corruption, Apple Photos opacity, digiKam edge cases). Three structural defenses:

1. **Categorize before applying.** Every delta gets confidence + category. Below `0.95` confidence → marked for executive review or user resolution, not silently applied. Old DAM tools had no model for "I'm not sure."
2. **Audit log of every action.** Every action — applied or proposed — writes to `journal.jsonl`. Bad calls are recoverable by replay (within the reversible-action subset).
3. **Patterns in a playbook, not in code.** Intelligence lives in `reconcile-imports.playbook.md` — readable, auditable, improvable. New failure modes extend the playbook.

Agentic failure modes acknowledged: over-confident inference (defense: conservative thresholds + sa.skeptic adversarial reviews of the playbook); hallucination (defense: every action grounded in evidence — hash, path, mtime); concurrency races (defense: exclusive lock at `.tropo-studio/reconciler/.lock`); substrate evolution (defense: playbook versioning).
